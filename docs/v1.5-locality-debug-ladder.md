# v1.5 — chat 路径 locality 塌缩单变量阶梯定位 (debug plan)

> 依据: `docs/horen_debug/exp_debug_design.md`
> 类型: **measurement-only** · 只定位、不修复
> 目标: 用 S0→S3 单变量污染阶梯,定位 chat 路径 locality 塌缩(JQ-fact key 挤进 ~0.88 窄锥、PARA≈NEG)**第一次在哪一级崩** = 真凶。

## 背景锚点(别再质疑)
- stock HoReN 在 ZsRE 上 locality=1.0(N=30 sanity 已验,见 `docs/zsre-stock-sanity-N30.md`)→ editing backend 健康,**锅 100% 在 Engram chat/keying 层**。
- ZsRE src 本身也是高重叠模板,HoReN 在相似 key 上 locality 仍 1.0 → 我们的塌缩是**管线偏离**,不是 HoReN 缺陷。
- 上轮 pooling sweep(v1.4):query-span 内换 %(flat/40/60/80/last)**不是杠杆**,只整锥平移,PARA 与 NEG 始终黏一起。
- 两端点:**正确端** = ZsRE 配置(raw prompt + 60% pool + 无 scaffold + 异质主语)= locality 1.0;**坏端** = chat 配置(query-span flat-mean + chat scaffold + JQ 共享框架)= 0.88 锥塌缩。中间隔 3 个变量。

## S0–S3 阶梯(每级只改一个变量,其余严格固定)
| level | 改了什么(vs 上级) | fact src | belief src | 期望 |
|---|---|---|---|---|
| **S0** 基线 | ZsRE 原版:raw prompt + 60% pool + 无 scaffold | ZsRE 式独立问句 | 剥 JQ 的独立问句 | locality 好(margin 大) |
| **S1** +JQ 框架 | 只给 fact 加回 `of JQ's` | `What is the name of JQ's cat?` | 仍 flat | Δ=JQ 共享框架伤害 |
| **S2** +query-span pool | 60% → chat 的 query-span flat-mean | 同 S1 | 同 S1 | Δ=池化额外伤害(预期小) |
| **S3** +chat scaffold | raw → 完整 chat 模板(system + 空 RAG 窗 + query) | 同 S2 | 同 S2 | =现状,应复现 0.88 锥 |

## 测试集(10 条,从 `eval/samples.json` 按 id 捞真样本,禁手编)
- **5 条 JQ-fact**(F1–F5,共享 `JQ` + `of JQ's` 框架 → 最易撞车 → 主测):cat=Coco / car=Ford / 过敏 peanuts / dentist Dr.Lee / 住 Maple St。
- **5 条 belief**(B1–B5,剥 JQ 的 flat ZsRE 断言 → 对照):best team France / SF summers cold / Rust best lang / pasta terrible / capital Oakhaven=Vaelor。
- 每条字段:`{id, edit_text, src, paraphrases:[1-2], target}`,全英文。从 A 族(fact)/ X 族(belief)按 id 捞。

## 每条三个 probe(locality 判据,生产 deferral cosine,阈值 0.85)
- `DIRECT` 问自己的 src → 应触发(高) ← 编辑生效?
- `PARA` 问 paraphrase → 应触发(高) ← 泛化?
- `NEG` 问**同组内另一条**的 src(F1 的 NEG=问 F2;B1 的 NEG=问 B2)→ 不应触发(低) ← ★locality
- **健康** = DIRECT 高 ∧ PARA 高 ∧ NEG 低 ∧ **margin=PARA−NEG 大**;**病** = PARA≈NEG。★看 margin 不看绝对分(整锥平移会骗人)。

## 执行步骤(对应 design §6)
1. **READ/recon(并行)**:定位并报告 4 处真实代码位置 —— ① raw ZsRE keying(60% pool / `query_selection_strategy` / `_select_query`);② chat keying(query-span flat-mean + scaffold `_hero_render`);③ 生产 deferral cosine 算法 + 阈值 0.85;④ 复用的生产 keying 函数(`pool_span_rows` / `compute_key` / `query_span_in_rendered`)。**报告后再动手**。
2. **捞样本**:从 `eval/samples.json` 按 id 捞 10 条,构造上面字段,打印确认非编造。
3. **写 measurement spike**(scratchpad / spikes):实现 S0–S3 四级 keying 开关,每级对 10 条跑 DIRECT/PARA/NEG,算 margin。**复用生产 keying 函数,零 drift**。
4. **inline 串行跑**(单 GPU,独立 model 副本),出跨级表 + fact/belief 分组。
5. **STOP 报告**:四级表 + locality 第一次崩在哪级 + 回答 design §4 四问。**等用户定下一步,不自动进入修复**。

## 输出(rule 3,跑完补到本 doc)

## 结果(开发后)

### 落地 & 校验
- spike:`spikes/spike_v15_locality_ladder.py`(measurement-only,独立 model 副本,**未碰** server/codebook/memory/editing)。
- **生产路径零漂移**:`HOREN(cfg, model)` 装 adapter(layer-29 down_proj),config 从**真 yaml** 载(`normalize=true / qsel=last_60_perc_prompt_tokens_avg / qspan=flat / Hopfield β20 α0.1 iter1 / thr0.85`)。打分用 `keying.score`(生产 `_query` gate)。**drift-guard 通过**:自写 S0/S3 extractor == 生产 `compute_key(templated=False/True)`(`allclose atol 1e-3`)。
- 10 条真样本(fact `A0001–A0005` / belief `A0139–A0143`,按 id 捞)。S0 de-JQ = `JQ→{Riley,Jordan,Casey,Morgan,Quinn}` 机械 token swap(只换专有名词、不改句式,已逐条打印)。写/存 key = `key(edit_prompt)`(cloze 生产臂);probe DIRECT=queries[0] / PARA=queries[1] / NEG=同组 sibling 的 DIRECT。
- `torch 2.8.0+cu128 / True` 首尾一致;VRAM 16.2G;exit 0,无 traceback。

### 判读(§4 四问)— ⚠️ 数据推翻了预判
1. **S0 基线 OK**:fact/belief 均 **0/20 误触发**(NEG 地板 0.809/0.813 < 0.85),locality 完好 → 测量管线没问题,继续。(margin 全程偏薄,因表示本就挤在 ~0.81–0.92 窄 cosine 带;**locality 真信号是 NEGxfire 是否过阈,不是 margin 绝对值**。)
2. **locality 第一次崩在 S2(pooling),不是 S1**:fact NEGxfire `S0 0 → S1 0 → S2 4 → S3 11`。S1 加 JQ 共享框架 **毫无变化**;S2 换成 query-span flat-mean 池化才首次误触发;S3 加 scaffold 急剧恶化。
3. **fact/belief 在 S1 不分叉(预判落空)**:S0→S1 两组 margin 纹丝不动(fact +0.037→+0.037,belief +0.070→+0.070)。**「共享 JQ 框架是真凶」被证伪。** 真分叉在 S2/S3:chat keying 下 fact 比 belief 崩得更狠(S3 fact 11/20 vs belief 7/20),但触发器是 pooling+scaffold,不是 JQ 数据。
4. **一句话结论:真凶 = chat keying 本身(flat-mean 池化 ⊕ chat scaffold)把整锥抬过固定 0.85**,不是 JQ 共享框架。跨 S1→S3 整锥单调上移(DIRECT 0.851→0.915、NEG 地板 0.815→0.898 一起涨,NEG 涨幅更大去贴 DIRECT),固定 0.85 落进锥内 → siblings 误触发。即 [[horen-key-cone-collapse]] 的「表示塌缩 + 单一固定阈值」,抬锥者精确定位到 **scaffold(S2→S3:4→11,单步最大)+ flat-mean 池化(S2)**,并**排除** JQ 框架(S1)与 v1.4 已排除的 intra-span %-旋钮。

### 与 v1.4 / 旧诊断的关系
- v1.4「intra-span %-旋钮(flat/40/60/80)不是杠杆」**仍成立**;本轮动的是**算子类**(raw `_select_query` last-60%-of-full-prompt → chat `_pool_span` flat-mean-of-span)+ scaffold,二者**确实抬锥**。
- 旧诊断「scaffold/JQ-frame 锥」里 **JQ-frame 部分被证伪**;锥的抬升来自 **scaffold + flat-mean 池化**(scaffold 为单步最大贡献)。统一图景:**语言(旧发现)与 chat-keying 构造(本发现)都在把同一窄锥滑/抬过那一个固定 0.85 阈值** → 修的是锥的绝对位置 / 固定阈值,不是 JQ 框架。
