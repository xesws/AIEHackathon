# v0.6 — 前端初步开发 + fake smoke test

对齐 `docs/frontend/visual_rcs/DESIGN.md`(保守版:**前端交互为实、后端象征性留白**)。
目标:把 `frontend/index.html` 从占位 stub 变成一个**能跑的纯前端交互原型**,并加一个
不依赖真实后端、不依赖浏览器自动化的 **fake smoke test**。

## 范围(有意克制)
- 只做前端交互逻辑 + mock 数据。**0 网络请求、0 真实后端依赖**(对齐 DESIGN §0)。
- 不引入 Node 工具链(本机无 node/npm,且 DESIGN §5 明确"无编译")。
- 不接 `serving/app.py`(其端点仍为 stub);所有"记忆动作"都是 React state 内部搬运。

## 环境事实(已探测)
- 无 node / npm / npx / pnpm / yarn;无本地浏览器;无 playwright / selenium。
- 有 Python 3.12 + pytest 9.1.1;`esm.sh` 可达(HTTP 200)。
- 结论:走 **no-build CDN** 方案;smoke test 用 pytest(serve + 静态断言),render 验证靠人工开浏览器。

## 方案 — no-build 单页
- `frontend/index.html`:Tailwind Play CDN + import map(react / react-dom / lucide-react 走 esm.sh)
  + Babel Standalone 在浏览器里转译 JSX(对齐 DESIGN §5 "无编译,arbitrary class 不可用")。
- `frontend/src/engram.jsx`:`UI_design_visual_plan.jsx` 的**可运行副本**(设计稿仍留在 docs/ 作 artifact)。
  仅三处改动:加 CDN 头注释 + `createRoot` 导入;`export default function Engram` → `function Engram`;
  文末挂载 `createRoot(#root).render(<Engram/>)`。组件主体逐字不动。
- 运行:`python3 -m http.server -d frontend 5173` → 浏览器开 `http://localhost:5173/`。

## fake smoke test — `tests/test_frontend_smoke.py`(pytest)
无 node/浏览器,做不到真·render;改为"工件可服务 + 交互逻辑已接线 + 纯前端契约"三层断言:
1. **serve**:`http.server` 起在临时端口,GET `/` 与 `/src/engram.jsx` 均 200。
2. **index.html 结构**:含 `id="root"`、import map(react / lucide-react / react-dom)、tailwind CDN、
   babel standalone、引用 `./src/engram.jsx`。
3. **交互接线(mock 数据)**:10 个 state(surface/dev/ragOn/editOn/input/justCommitted/messages/weights/buffer/refs)、
   curation 处理器(consolidate/burnOne/demoteOne/discardOne/editPending/burnAll/send)、
   三层搬运(setWeights/setBuffer/setRefs)、mock 种子(w1/w2/w3、b1/b2、r1/r2)、挂载(createRoot + render<Engram)。
4. **"fake"保证(DESIGN §0)**:组件源码**不得**出现 `fetch(` / `XMLHttpRequest` / `axios` /
   `localStorage` / `import.meta.env` / 后端端点(`/chat`、`/consolidate`、`/memories`)——
   证明这是纯前端 mock、后端只是留白。
5. **结构 sanity**:`engram.jsx` 大括号/圆括号/方括号配平;`function Engram(` 存在。

## 局限(诚实记录)
- 这是 **静态 + serve smoke**,不是 render smoke:断言"工件载入正常、交互逻辑已接线、零后端调用",
  但**不**执行 React 渲染(本机无浏览器)。真·渲染/交互需人工开 served URL 验证。
- 运行时仍依赖 CDN(esm.sh / tailwind / babel)联网;离线则页面空白(smoke test 本身不联网)。

## 产物
- `frontend/index.html` — no-build 单页(从占位 stub 替换):Tailwind CDN + import map + Babel Standalone,引用 `./src/engram.jsx`。
- `frontend/src/engram.jsx` — Engram 组件可运行副本(8 组件 + mock 状态 + 挂载),源自 `UI_design_visual_plan.jsx`。
- `tests/test_frontend_smoke.py` — fake smoke test(pytest,10 用例)。

## 结果
- `pytest -q tests/test_frontend_smoke.py` → **10 passed in 0.67s**。
- `pytest -q tests/`(全量回归) → **91 passed in 2.27s**(81 原有 + 10 新增,无回归)。
- 覆盖断言:
  - serve:`/` 与 `/src/engram.jsx` 均 HTTP 200;index.html 含 root/importmap/tailwind/babel/engram 引用。
  - 接线:8 组件定义、10 个 useState、7 个 curation 处理器、三层 setter + burnAll 搬运、mock 种子(w1/w2/w3、b1/b2、r1/r2)、`createRoot…render(<Engram>)`。
  - **零后端/网络**:组件源码不含 `fetch(` / `XMLHttpRequest` / `localStorage` / `import.meta.env` / `http(s)://` / `/chat`·`/consolidate`·`/memories` —— 印证 DESIGN §0 的纯前端 mock 契约。
- 人工 render:本机无浏览器,未执行;在有浏览器的机器上 `python3 -m http.server -d frontend 5173` → 开 `http://localhost:5173/` 即可验证(需联网取 CDN)。
