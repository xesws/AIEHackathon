## 目标
逻辑层(`ingest` / `triggers.manual` / `consolidate.run_pass` / `generate` / `store`)已全部写好且
在真权重上验过(v0.5.1 / v0.5.2 全绿)。`serving/app.py` 仍是 3 个 `raise NotImplementedError` stub。
本步把 app.py 写成一层 **HTTP 搬运壳**:接 HTTP → 调已验证的 memory/generate 函数 → 回 JSON,

★ 核心原则(INV-S1):serving 不含业务逻辑,只搬运。serving → memory 单向,绝不改被调函数。

## 契约(照搬 serving_design.md §4,字段名不得改)
- **POST /chat** `{message:str, rag_off:bool=false}` → `{reply:str, buffer_count:int, learned:list}`。
  行为(顺序固定):① `ingest([{"role":"user","content":message}])`(edit→buffer / rag→rag_store);
  ② `rag_hits = [] if rag_off else rag_store.search(message, k=5)`;
  ③ `reply = generate(message, model=current_model(), buffer=buffer.load_unconsolidated(),
  rag_hits=rag_hits, use_chat_template=True, tok=tokenizer())`。**ingest 在 generate 之前**(写入即读)。
- **POST /consolidate** `{}` → `{n_written:int, buffer_count:int}`。行为:`triggers.manual()`(= run_pass)。
- **GET /memories** → `{buffer:[...], consolidated:[...], counts:{buffer:int, consolidated:int}}`。
  读 `store.by_status("buffer"|"consolidated")`,经 `schema.to_dict` 序列化(只读)。

## 关键签名事实(探查已定,照搬,勿臆测)
- `consolidate.set_model_provider(fn)` 取**零参 callable**(不是 model);startup 须
  `set_model_provider(lambda: model_host.current_model())`。
- `ingest(chat)` 收 `[{role,content}]` **列表**(非字符串)→ app.py 把 `message` 包成单条 user 消息。
- `generate(...)` 中 `model=` 必填、Plan B chat 分支须 `use_chat_template=True`、`tok=None` 会回退到
  `model_host.tokenizer()`。
- **★rag-off 写法**:`with_rag` 同时门控 buffer 窗口与 rag 窗口。rag-off = 传 `rag_hits=[]` 且
  **保持 `with_rag=True`、仍传 `buffer=load_unconsolidated()`**;**绝不** `with_rag=False`(会连用户事实窗口一起清,违 INV-S3)。
- `current_model()` 返回常驻 **HF model**;startup `load_base()` ~60–90s、~16GB(静默,无 timing 打点)。
