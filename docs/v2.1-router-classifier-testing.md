# v2.1 — Router Classifier Testing(分类器准确率门)

> 本轮目标(一句话):给"自动路由"建一个**可回归、带 fact→belief 不对称安全门**的轻量测试,
> 把当前只压在 `extract.py` 一次 LLM `type` 判定上的路由对错变成可测量、可回归的门。
> **只测量,不调优。**

---

## 一、计划节

### 1. Context / 为什么做
v1.9 把路由从「by-shape 分类器 + router 级 `CONF_MIN=0.5` 安全垫」换成纯确定性映射
`route = "edit" if item.type == "belief" else "rag"`(`memory/router.py:25`),并在 git `81bdfd9`
里**删掉了 router 级 confidence 安全垫**(原来"分类器不确定就回退到可逆 `rag`")。后果:**全部路由对错
都压到了上游 `extract.py` 那一次 LLM 的 `type` 判定(fact/belief/other)上**,而 router 这层再也没有
"不确定就走安全方向"的兜底。

当前对这唯一关键判定的覆盖只有 `spike_v19` Part A 的 9 条 soft report(`9/9`),**非 gate、无边界 case、
无回归**。v2.1 把它升级成可回归、带不对称门的测试。

**下游已覆盖、本轮不重测/不改**:`router.route`(`tests/test_router.py` 6 绿)、写读管线
(`tests/test_fact_belief_routing.py` 5 绿)。★ 唯一被测对象 = extract 的 LLM `type` 判定。

**本轮约束**:数据集**只用英文**;不追求严谨,抓大错即可;少来回、不复杂。

### 2. 标注口径 rubric
标注单元 = 一句英文 teach 输入,标 `gold_type ∈ {fact, belief, other, dropped}`,非 dropped 再标 `expected_route`。

- **fact** = 用户 / JQ 的**客观、可验证的个人属性**(猫名、过敏、住址…)→ route `rag`。
  *观点 frame 不翻案*:`"I believe my cat is named Coco"` 底层是客观属性,仍是 **fact**。
- **belief** = 用户的**主观观点 / 偏好 / 习惯 / 世界观** → route `edit`。
  *习惯句*(如 `"I default to Postgres for OLTP"`)标 **belief** —— **对齐生产 few-shot**(`extract.py:71-79`
  把 Postgres 习惯标成 belief)。gold 必须与 few-shot 口径一致,否则测试自相矛盾。
  ⚠️ **此口径我标为存疑,留给 v2.2**;本轮不改 prompt。
- **other** = **不关乎用户本人**的耐久信息(日程、参考内容)→ route `rag`。
- **dropped** = 临时 / 闲聊,extract 期望**不产出任何 item**。代码里**没有确定性 ephemeral 过滤器**
  (`extract.py` 无临时态 gate,纯靠 system prompt 让 LLM 自行抑制),故 dropped 最抖,Layer B 里按 **SOFT** 处理(只报不门)。无 route。

映射(由真 `router.route` 推导):belief→edit;fact / other→rag;dropped→无。

### 4. Layer A — 离线 pytest(CI-safe)
`tests/test_router_classifier_dataset.py`,**不调 LLM**,沿用 `test_router.py` 风格。断言:
① 可加载、size 合理;② `gold_type` 合法、`id` / `text` 唯一;
③ 非 dropped:`expected_route == router.route(MemoryItem(type=gold_type, …))`(耦合生产映射);
④ dropped 无 `expected_route`;⑤ seed 9 条在集内;⑥ 各 gold_type + 关键边界 category 覆盖。

### 5. Layer B — 真 LLM 评测(opt-in,不进 CI)
`spikes/spike_v21_router_classifier_eval.py`,standalone spike。`.env` 随 `memory.llm` 自动加载,**无需 GPU**(仅走 OpenRouter extract)。
- 记录被测 model = `memory.llm.DEFAULT_MODEL`(写进 `R["model"]`),`N=3`。
- 每条跑 `extract.extract` 3 次,空 → `"dropped"` 否则 `items[0].type`,取 **majority**;记 jitter。
- 产出:**4×4 混淆矩阵**、overall accuracy、per-class precision/recall、**fact→belief 危险计数**。
- **不对称门(coarse)**:`HARD = (majority 下 fact→belief 混淆 == 0)`,破门即 FAIL;
  **SOFT**:overall accuracy(参考阈 ≥0.85)、belief→fact(宽容)、dropped 检出 —— 只报不门。
- 末尾:`gates / overall`、`"="*78` VERDICT、`MACHINE_RESULT <json>`,`return 0 if overall else 1`。

### 6. 护栏
只新增 4 个文件(doc + dataset + Layer A + Layer B),**绝不 `git add -A`**;不碰 `extract.py` 的 prompt/few-shot/
`CONF_MIN`、不碰 router/schema/llm/serving/frontend/eval/`samples.json`/现有测试。若 HARD gate FAIL = **有价值发现**
(无安全垫时自动路由不安全)→ 报告 + 建议 v2.2,**本轮不修**。

## 二、结果节

**运行环境**:本轮中途机器重启,易失盘(`/usr/local/lib/.../site-packages` 等)上的 pip 包丢失;
按"**排除 torch**"重装了缺失依赖(pytest / openai / python-dotenv + transformers / fastapi / uvicorn /
pymongo / sentence-transformers),**torch 全程保持 `2.8.0+cu128`、CUDA 可用**(护栏自始至终满足)。
GPU 现为 A40(driver 550 / CUDA 12.4,cu128 向下兼容);本轮 Layer B 不用 GPU。

### 1. 跑法回顾
数据集 16 条(English-only,9 条 v19 seed + 7 条边界);Layer A 离线 pytest **7/7 绿**(零网络);
Layer B `spikes/spike_v21_router_classifier_eval.py` 真 LLM 跑通 **exit 0**。被测 model = `qwen/qwen3.7-max`,N=3。

### 2. 关键数字
| 指标 | 值 |
|---|---|
| overall accuracy | **1.0 (16/16)** |
| ★ HARD fact→belief 泄漏(majority) | **0** ✅ |
| fact→belief 原始单次跑泄漏 | **0 / 18** |
| belief→fact(SOFT,可容忍) | 0 |
| dropped 检出(SOFT) | 2/2 |
| seed sanity(对齐 v19 9/9) | **9/9** ✅ |
| jitter(N=3 非一致条目) | **0/16**(全 unanimous) |
| per-class P/R(fact/belief/other/dropped) | 全 1.0 |

### 3. 混淆矩阵(rows=gold, cols=pred-majority)
| gold＼pred | fact | belief | other | dropped |
|---|---|---|---|---|
| **fact** | 6 | 0 | 0 | 0 |
| **belief** | 0 | 5 | 0 | 0 |
| **other** | 0 | 0 | 3 | 0 |
| **dropped** | 0 | 0 | 0 | 2 |

完美对角,无任何混淆。

### 4. 边界 case 表现(本轮价值所在)
- ★ **fact 伪装成观点**(`I believe my cat is named Coco` / `I think I live on Maple Street`)→ **3/3 都判 fact,零泄漏**。HARD 门核心压力点,通过。
- **习惯句**(`I default to Postgres for OLTP`)→ belief 3/3,**与生产 few-shot 口径一致**(本轮按约定不改;见 §6)。
- **coref / 人称混写**(`I'm JQ and my cat is named Whiskers`)→ fact 3/3。
- **临时态**(`I'm waiting in line` / `feeling tired`)→ **dropped 3/3**(LLM 自行抑制,虽无确定性过滤器,本轮未抖)。
- **无边界 case 失败。**

### 5. 诚实结论:自动路由够不够可靠?
- **在 `qwen/qwen3.7-max` + 这 16 条英文上**:分类器表现**完美且稳定**(0 jitter),最关键的**危险方向 fact→belief 零泄漏**,
  自动路由在该样本上是安全的。HARD 门已成为**回归 tripwire**:今后换模型 / 改 prompt 若引入 fact→belief 泄漏,本 spike 会 FAIL。
- **但不要过度外推**:① 样本小(16)、英文、且按用户要求"不追求严谨",100% ≠ 野外无泄漏;
  ② 准确率**随模型变**——弱模型可能泄漏,换 `OPENROUTER_MODEL` 必须重跑本门;
  ③ dropped 无确定性过滤器,本轮不抖含运气成分,长期仍按 SOFT;④ habit→belief 口径本身存疑(见 §6)。

### 6. 是否需要 v2.2 / 建议
本轮**未触发**"必须立刻恢复安全垫"的证据(HARD 门干净)。但代价不对称依旧存在,建议 v2.2(择机、非紧急):
1. **扩样**:更多 fact-伪装、真实(非虚构)世界 belief、多句 / 长句、中文(本轮按用户排除),压泛化。
2. **复核 habit→belief 口径**:行为习惯进权重是否真合理(`extract.py:71-79` few-shot),还是更该走 RAG —— rubric 已标的存疑点。
3. **defense-in-depth(可选)**:即便满分,考虑给 router 加回一个 confidence 安全垫(不确定 → 走可逆 rag),
   把"危险方向"从"靠分类器永不犯错"降级为"犯错也安全"。

**本轮按约定只测量,未改任何 `extract` 的 prompt / few-shot / `CONF_MIN`。**
