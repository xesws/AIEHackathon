# v1.9 — fact/belief 分流重构(实现计划 + 结果)

> 设计来源:`docs/horen_debug/rebuild_design.md`。本文是该设计的**实现计划**(开工前写,
> CLAUDE.md 规则 4)+ **结果记录**(收尾补,规则 3)。
> 范围:routing + 写入路径 + 读取路径(prompt)。**不碰 eval / samples.json / data**。

## 0. 核心决策(锁定,实现不动摇)

| 信息类型 | type | route | 去向 | 理由 |
|---|---|---|---|---|
| JQ 客观属性 | `fact` | `rag` | rag_store | editing 会 sibling 互串;RAG 强项 |
| JQ 主观看法/偏好 | `belief` | `edit` | buffer→consolidate→weights | 天然分得开;是 proof 主角 |
| 与 JQ 无关的重要信息 | `other` | `rag` | rag_store | long-content,本就该 RAG |

**★ 最关键认知(决定 prompt 结构):belief 是隐式的(在权重里,forward 时生效,不是文本,
无法 retrieve 填进 prompt)。所以 prompt 里只有 fact/other 段(显式 RAG 文本),没有 belief 段。**

## 1. 不变量(INV)

- INV-1 fact 永不进 buffer/editing。
- INV-2 belief 永不进 rag_store。
- INV-3 prompt 里没有 belief 段(belief 隐式在权重;显式段只有 fact/other)。
  - buffer-seg 仅作 **un-consolidated belief 的过渡显示**(暂存期可见),consolidate 后抽干 →
    proof 时该段为空,prompt 无 belief 文本。
- INV-4 proof:consolidate 后 + buffer 抽干 + rag_off → belief 仍答对。
- INV-5 route 由 type 决定,type 由 LLM 判(extract 那步);下游看 route 不看 type。

## 2. 模块改动清单(本轮实际)

| module | 改什么 |
|---|---|
| `memory/schema.py` | `MemoryType` 收敛为 `{fact, belief, other}`;`from_dict` valid_types 同步 |
| `memory/extract.py` | LLM type 词表 + system prompt + few-shot 改为 fact/belief/other(判据见设计 §2.1)。**这是 INV-5 的 LLM 判定点** |
| `memory/router.py` | `route()` 从 by-shape 改为 **deterministic type→route map**:fact/other→rag,belief→edit。删除旧 shape 分类器 |
| `serving/ingest.py` | 不改逻辑(已按 `it.route` 分流);确认 fact/other→rag_store、belief→buffer |
| `memory/rag_store.py` | 确认装 fact+other 且保留 `type` 标签;短句可检索(单 chunk)。基本不改 |
| `memory/consolidate.py` | 不改(输入只剩 belief) |
| `memory/prompt.py` | **★重构**:rag_hits 按 type 拆成 fact 段 + other 段;buffer 段保留为过渡;**无 belief 段** |
| `generate.py` / `serving/app.py` | 不改(签名不变,拆分在 prompt 内部完成) |

## 3. prompt 结构(token_budget=None hero 路径)

```
[SYSTEM]
[Known facts about the user — adopt by default]     ← rag_hits where type=="fact"
[Reference material — does not override]            ← rag_hits where type!="fact"(other)
[Pending unconsolidated memory — adopt by default]  ← buffer 整段注入(过渡 belief;consolidate 后为空)
[history]
[user query]
```

budget 路径优先级:buffer(过渡,最高)> facts > history > others(docs,最先裁)。每段 header 恒渲染(INV-5)。

## 结果记录

### 单测(CPU)
- 全套 `pytest tests/` → **162 passed**(baseline 159;net +新增 - 旧 router shape 测试)。
- 新增 `tests/test_fact_belief_routing.py`(5 条):断言 A/B/D/E + INV-3(monkeypatch LLM/editing,真 embed)。

### E2E proof(真模型,`spikes/spike_v19_fact_belief_proof.py`,exit 0)
- **Part A — 真 LLM 分类器(§7 9-条手验):9/9 全分对**(belief→edit / fact→rag / other→rag)。
- **A 路由**:rag_store=[fact×3, other×3] 带 type 标签;buffer=[belief×3];INV-1/INV-2 成立。✅
- **C ★命门 — belief 在权重**:consolidate 后 buffer 抽干 + rag_off,3 条 belief 全部答对
  (Vaelor / Zarithon / Mount Brindlewick;sim 0.94–1.00 > 0.85 阈值);pre-edit baseline 全答不出 → 归因干净(真编辑,非先验)。✅
- **B fact via RAG**:cat/peanuts/Maple 全答对,且 fact 文本出现在 prompt 的 FACT 段。✅
- **D other via RAG**:Nov 15 / ops vault / 12 steps 全答对。✅
- **E 不串**:cat query 检索 top1 = "JQ's cat is named Coco"(无锥塌缩)。✅
- **INV-3**:proof 时 prompt 无 belief 文本、无 belief 段。✅
- consolidate:n_written=3,buffer_after=0,23.3s。**显存 16.5 → 16.9 GB**(48GB 卡余量充足)。无报错。

### 结论
分流 thesis 在真模型上成立:**fact 走 RAG(prompt 里看得见)、belief 内化进权重(prompt 里看不见但答对)**,
绕开了 sibling 锥塌缩。这正是 §0.3 的 demo proof 可视化。下一步:前端交互组件落地该对比。

> 小注:belief 的贪婪续写有重复退化("Zarithonononon…"),因 edit 命中极强 + `use_cache=False`;
> target 已清晰命中,serving 用 `max_new_tokens=160` + chat EOS 真聊天回复不受影响。非阻塞项。
