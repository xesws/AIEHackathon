# v2.4 — 修复 RAG badge 错配("每个答案都显示『检索自你的文档 · 用户的猫名叫 Coco』")

> 本轮目标(一句话):稀疏 store 下 `search()` 永远返回 top-k(无相关性 floor),前端 RAG badge 又
> 无条件挂 `retrievedDocs[0]` → **每个 rag-on 答案都误挂"检索自你的文档·猫"**。本轮用**方案 C**
> (后端 additive 透出 score + 前端 floor 闸)修这条**显示路径**,**不碰** attribution / 权重 badge / 检索排序选条。

---

## 一、计划节

### 1. 根因(两层叠加,别误诊)
- **后端** `memory/rag_store.py:116` `search()` → 末尾 `return candidates[:k]`,**无相关性 floor**:
  只要 `_chunks` 非空就返回 top-k,**不管多不相关**。稀疏 store(demo 只有 猫/过敏/会议 几条)下,
  **任何** query 的 top1 都会是"猫"。
- **前端** `frontend/src/engram.jsx:501-503`,RAG badge 只要 `retrievedDocs.length>0` 就挂、
  **无条件**显示 `retrievedDocs[0]` → 每个 rag-on 答案都显示"猫",**不看相关性、不看答案到底用没用它**。

★ **bug 不在 `attribution()`**(权重侧,有 `0.85` 阈值,是健康的)。本轮**只修 RAG 检索显示**这一条路;
**绝不碰** `attribution()`(`serving/app.py`)和**权重 badge**(`engram.jsx:506-509`)。

### 2. 修法 = 方案 C(后端透 score additive + 前端 floor 闸)
1. **`memory/rag_store.py` — `search()` 加可选 `with_scores=False`(additive)**:
   - `with_scores=True` → 返回 `list[tuple[MemoryItem, float]]`,`float` = 该 parent 的 **best-chunk cosine**
     (`search` 内部 `best_for_parent` **已经算出来了**,只是没透出 → 直接暴露)。
   - 默认 `False` → **返回类型/排序/选条/top-k 全不变** → 现有 caller / 测试全绿。
   - ★ **不改排序、不改选哪些条、不改 top-k 数量** —— 只多透一个 score 字段。

2. **`serving/app.py` — `/chat` 用 `with_scores=True`**,把 score 带进 `retrieved` 每一项:
   ```python
   rag_scored = [] if payload.rag_off else rag_store.search(message, k=5, with_scores=True)
   rag_hits   = [h for h, _s in rag_scored]               # generate 仍吃纯 items
   ...
   "retrieved": [{**schema.to_dict(h), "score": round(s, 4)} for h, s in rag_scored],
   ```
   (`rag_off` 分支仍 `rag_scored=[]` → `retrieved=[]`,不变;`generate` 收到的 `rag_hits` 仍是纯 items,prompt 不变。)

3. **`frontend/src/engram.jsx` — RAG badge 加 floor 闸**:
   - `send()`(~712):`docs = (resp.retrieved||[]).map(d => ({ text: d.text, score: d.score }))`
     —— `retrievedDocs` 从 `string[]` → `{text,score}[]`(全仓仅 badge 一处消费它,同步改)。
   - badge(501-503):**仅当存在 score > `RAG_FLOOR` 的 retrieved 时才挂**,且显示**那些 above-floor 的** text;
     否则**不挂 RAG badge**。
   - **不动** seed/mock 那条(`m.retrieved`,~496-500)、**不动**权重 badge(~506-509)。

### 3. ★ FLOOR 经验标定(headless 可做,实测不拍脑袋)
一次性脚本对 demo 样本 store 实测两个 cosine(走真 `search(..., with_scores=True)`):
- (a) `"What is the name of my cat?"` → 猫这条 score(应**高**);
- (b) 无关 query `"best place to stay in winter"` → 猫这条 score(应**低**)。
FLOOR 取两者之间(MiniLM `all-MiniLM-L6-v2`,经验 ~0.3 附近,**以实测为准**)。
两数 + 选定 FLOOR 写进结果节;验收口径:cat-query 命中猫、无关 query 不命中。

## 二、结果节

> **结论:方案 C 落地,所有回归绿。** 后端 additive 透 score(默认行为字节不变)+ 前端 floor 闸;
> FLOOR 实测标定 = **0.3**;`pytest tests/` 169 绿;真后端 `spike_v22` 8 门仍 OVERALL PASS。
> ⚠ "无关 query 的 badge 真消失"那一帧需浏览器,留后续手测(本轮只保证代码对 + score 标定合理 + 回归绿)。

### 1. ★ FLOOR 经验标定(实测,走真 `search(..., with_scores=True)`)
demo store = `{猫, 过敏, 会议}` 三条;对每个 query 取**猫这条**的 best-chunk cosine:

| query | 猫 score | 同 query 的真·相关条 | 判定 |
|---|---|---|---|
| **(a)** `What is the name of my cat?`(相关) | **0.6403** | — | 猫应命中 ✅ |
| **(b)** `best place to stay in winter`(无关) | **-0.0129** | — | 猫不应命中 ✅ |
| (extra) `When is the Q3 board meeting?` | 0.0276(below) | 会议 **0.8419** | 只命中会议 ✅ |
| (extra) `What am I allergic to?` | 0.0445(below) | 过敏 **0.5060** | 只命中过敏 ✅ |

- **相关簇 ≥ 0.50,无关簇 ≤ 0.045**,中间是干净的空档;midpoint(a,b)=0.3137。
- **选定 `RAG_FLOOR = 0.3`**(`engram.jsx`),落在空档内、略低于 midpoint(对"留住真命中"更稳)。
- 验收口径达成:cat-query 命中猫(0.64>0.3)、无关 query 不命中(−0.01<0.3);且 floor 对 会议/过敏 query
  也只放行各自真·相关条,**不会再误挂猫**。
