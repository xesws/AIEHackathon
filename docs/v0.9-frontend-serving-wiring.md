## 映射(接 serving 的三条)
- `send()` → `POST /chat {message, rag_off: !ragOn}`:append user → await → append assistant

- `consolidate()`(LabPanel "consolidate now")+ `onBurnAll`(MemorySurface "全部写入")→ `POST /consolidate {}` → `refresh()`(同一 async)。
- `refresh()` = `GET /memories` → `weights ← consolidated.map({id,text})`、`buffer ← buffer.map({id,text,status:"new"})`。mount + 每次 chat/consolidate 后。
- loading 态:`booting`(mount 轮询 /memories 直到模型就绪,显示"模型加载中…",一次性 readiness gate)、`sending`(/chat in-flight)、`consolidating`(/consolidate in-flight)→ disable 控件 + 文案。

### 部署说明(RunPod:跨端口/代理)
浏览器不在 pod 上,`localhost:8077` 不可达。已处理:
- **后端必须 bind `0.0.0.0`**(非 `127.0.0.1`),否则 RunPod 代理/外部不可达:`uvicorn serving.app:app --host 0.0.0.0 --port 8077`(app.py `__main__` 已是 0.0.0.0)。
- **前端 `API` 自动探测**:RunPod 代理域名 `<podid>-<port>.proxy.runpod.net` 自动把端口子域换成 `-8077`(同协议同方案 → 无 mixed-content、无需手填);localhost 转发 / 直连 IP 则用 `host:8077`。可用 `window.__ENGRAM_API__` 覆盖。
- ⚠ **用户侧**:需在 RunPod 把 **8077 也作为 HTTP 端口暴露**(与 5173 同样),`<podid>-8077.proxy.runpod.net` 才存在。直接在浏览器开 `https://<podid>-8077.proxy.runpod.net/memories` 应返回 JSON;否则即 8077 未暴露。
