# v0.4.1 — `memory/` 组件做到 full tier(判断/质量维度;零基建)

> 阶段:把 v0.4 的 hero/minimal `memory/` 在**不需要外部基建**的能力维度上推到 DESIGN §4 的 full 描述。
> 性质:**开发前计划**(CLAUDE 规则 4),开发后在 §结果 回填。沿用 agent-team workflow。

## 1. 背景与范围
v0.4 已把 `memory/` 做到 hero tier 并验证(hero smoke token-EM=1.0;5 样本 reliability 5/5,locality 无泄漏)。
本轮把**判断/质量**维度做满。用户**明确纠正范围**:

- **不做 change-stream;不上 replica set;不碰 `serving/triggers`、`serving/app`。**
- **consolidate 触发只做 manual**("Consolidate Now" 直接 `run_pass`);threshold/timer 属 serving、本轮不做。
- **持久化保持内存 store**;真 Mongo / 本地落盘(SQLite/JSON)/ standalone Mongo 均**后置**(本文 §明确不做 记录)。
- 「100%」= 零基建的质量维度做满,不含 Mongo/持久化/触发器基建。

目标:各模块达 DESIGN §4 full(去掉 Mongo/change-stream 部分);**hero smoke + 样本 eval 不回归**。

## 2. 各模块 full-tier 工作(零基建)
| 模块 | full 要做 | 备注 |
|---|---|---|
| `schema.py` | `Decision.target_ids: Optional[list[str]]=None`(多目标 supersede,§10-11);`from_dict` 加 enum/类型强校验 | 基础,先行;向后兼容 |
| `dedup.py` | embedding-NN 召回 shortlist(≥阈值 top-m)→ **fast-LLM judge**(llm 缝判 same/changed/new)→ Decision;多近邻→`target_ids` | 保留阈值预筛;mock llm |
| `extract.py` | 输出 schema 强校验 + 每条**置信度**(阈值过滤)+ **coref 消解**(代词→实体)+ **多语言**(不强制英文)+ **批处理**(长 transcript 分段) | prompt 工程 + 轻代码 |
| `router.py` | 稳健 LLM shape 分类返回**置信度** + 阈值(不确定→rag);保留确定性 atomic 预筛 | |
| `rag_store.py` | **chunking**(长 text 切重叠块逐块 embed,块→父 item 映射)+ **re-rank**(cosine top-N 块→去重到父→**LLM re-rank**→top-k);仍内存 store | Mongo 持久化 deferred |
| `prompt.py` | **token 预算裁剪**(超预算按优先级裁,近似计数,纯函数)+ **多轮 history**(真正线程化 + 裁剪) | §10-4 |
| `consolidate.py` | **批量去重**(一次 embed 全候选+registry)+ **重试**(editing.edit 失败退避重试 N 次才留 buffer)+ 多目标 supersede(retire 全部 target_ids)+ 有序 best-effort 原子性 | 触发仍 manual;Mongo registry/change-stream 不做;edit 串行(单模型/单 codebook) |
| `buffer.py`/`store.py`/`llm.py`/`embed.py` | **不动**(re-rank/judge 复用 `complete`;embed 已批) | 持久化 deferred |

## 3. 明确不做(本轮 OUT,记为 deferred)

- **Phase 2(测试,并行)**:扩/加单测,mock 四点(llm/embed/editing/store)。
- **Phase 3(GPU)**:回归 hero smoke + 样本 eval(不退)+ rag chunk/re-rank 小检索验证。
- agent 只改各自文件、不装包、不跑模型/网络;保持已冻结公共签名不变(extract/router/buffer/rag_store/dedup.classify→Decision/consolidate.run_pass/prompt.build_prompt)。

## 8. 结果(开发完成)

> 用 agent-team workflow 实现:Phase 1(schema → 5 并行质量模块 → consolidate,7 agent)+ Phase 2(6 测试 agent)。全程 torch `2.8.0+cu128` 未动;零新增重依赖;INV-11 保持;向后兼容(老 15 单测全过)。

### 模块改动(全 backward-compatible)
| 模块 | full 落地 |
|---|---|
| `schema` | `Decision.target_ids`(多目标);`from_dict` 枚举/类型/ts 强校验 |
| `dedup` | embedding-NN shortlist(top-M=5)→ fast-LLM judge(duplicate/supersede/unrelated)→ Decision;多目标 supersede;LLM 失败回退 v0.4 阈值 |
| `extract` | 输出 schema 强校验 + 置信度门控(CONF_MIN=0.5,缺省=1.0)+ coref 消解 + 多语言 + 长文本分窗批处理去重 |
| `router` | LLM 分类返回 confidence + 门控(缺省=1.0)+ 不确定→rag;保留确定性 atomic 预筛 |
| `rag_store` | chunking(重叠块+块索引)+ cosine top-N→父去重→ LLM re-rank(失败回退 cosine);内存,store 不动 |
| `prompt` | 可选 `token_budget`+`count_tokens`(默认 None=字节级不变)+ 按 buffer>history>docs 优先级裁剪 + 段头恒在 + 省略标记 |
| `consolidate` | editing.edit bounded 重试(2+退避)+ 多目标 supersede(retire 全部);触发仍 manual,edit 串行 |
