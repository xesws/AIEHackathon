# v2.5 — 修复 CHAT 回复行为(F5 教学误当问答 / F6 复述内部 header / F2 "已经有了")

> 本轮目标(一句话):三个 CHAT 回复症状(F5 / F6 / F2)**同一个根因**——`memory/prompt.py`
> 的 `SYSTEM` 指令对"怎么回复用户"**零约束**:既没说别复述 memory window 的内部 section header,
> 也没区分"用户在陈述/教学"还是"在提问"。本轮**只改 `SYSTEM` 指令文本**(最小面),
> **不动**三段式骨架 / FACT·BUFFER·DOCS 角色 / fact·other 拆分 / INV-3(无 belief 段)/ token_budget 裁剪。

---

## 一、计划节

### 1. 三个症状(同一根因)
`build_prompt()`(CHAT 路径,`use_chat_template=True` 时走它)拼出的 system message =
`SYSTEM`(`memory/prompt.py:45-49`)+ 一个 memory window;window 里有**字面方括号 header**:
`[Known facts about the user — adopt by default]`、`[Pending unconsolidated memory — adopt by default]`、
`[Reference material — does not override]`。`SYSTEM` 对回复行为零指令,于是:

| 症状 | 现象 | 缺的指令 |
|---|---|---|
| **F6** | 模型把 `[Known facts...][Pending...]` 脚手架 header 原样吐给用户 | 没说"别复述/引用内部 header 与结构" |
| **F5** | 用户在**陈述/教学**(如 "best food is Mifen")时,模型把陈述当问题答 / 长篇大论(整段介绍米粉) | 没说"陈述→一句简短确认,别讲解展开" |
| **F2** | 用户教的 fact 已在 `[Known facts]` 里 → 模型答"我们已经有了" | 没说"已知的事也只简短确认,别说重复了" |

### 2. 根因定位(已读 `memory/prompt.py` 确认)
- `SYSTEM`(45-49 行)只描述了 memory window 的**内容语义**("known facts → adopt by default;
  reference material → 不 override"),**完全没**描述**回复风格**与**陈述 vs 提问**的处理。
- memory window 的三个 header 是**内部脚手架**(给模型看的 section 分隔),`SYSTEM` 没告诉模型
  "这些是内部上下文、绝不能复述给用户" → F6。
- `SYSTEM` 没有"用户在教学时简短确认"的规则 → 模型默认把每条输入当成要回答/展开的话题 → F5 / F2。

### 3. 修法 = 只给 `SYSTEM` 加 4 条简洁指令(最小面)
在保留原有两句(Engram 身份 + window 内容语义,**逐字不动**)的基础上,**追加**:
1. **(F6)** memory window 是你的**私有内部上下文**——**绝不**向用户复述、引用、或提及它的
   section header 或结构。
2. **(F5)** 当用户是在**陈述/告诉你**关于他自己的偏好或事实(**而不是提问**)时,用**一句**
   简短自然的话确认已记下(如 "Got it — I'll remember that."),**不要**讲解、展开、或反驳。
3. **(answer)** 当用户在**提问**时,**直接、简洁**地回答,相关时用 memory window 佐证。
4. **(F2)** 如果用户告诉你一件你**已经知道**的事,简短确认即可——别说"重复了 / 已经有了"。

★ **指令 #2 必须只对"陈述句"生效、不对"问句"生效**(措辞里显式写 "rather than asking a question"),
否则会压制对**问句**的回答 —— 见 §5 的 demo-proof 风险。

### 4. 硬性护栏(本轮严格遵守)
- **只改** `memory/prompt.py` 的 `SYSTEM` 文本字符串;**不动**:三段式骨架、
  `FACT_HEADER / BUFFER_HEADER / DOCS_HEADER` 的角色与文本、fact/other 拆分、INV-3(无 belief 段)、
  `token_budget` 裁剪逻辑(budget 数学通过 `_count(SYSTEM)` **自动适配**新长度)。
- **绝不碰**:`serving/`、`frontend/`、`generate.py`、`memory/extract.py`、`dedup.py`、`editing.py`、
  `keying.py`、`model_host`、β / 阈值 / 池化 / codebook、任何 `samples.json` / `demo_samples*.json`、`.env`。
- **`SYSTEM` 文本不得含单词 "belief"**(两处测试断言 `"belief" not in content.lower()`),
  也不得新增**整行被 `[...]` 包裹**的行(header_lines 断言只允许三个已知 header)。→ 故 `SYSTEM`
  保持为**无换行**的连续 prose,且用 "preference or fact about themselves" 表述、回避 "belief" 一词。
- 测试用例**只用英文**;**绝不** `git add` / `commit` / `push`;torch `2.8.0+cu128` 不动。
- **绝不 boot GPU**(不跑 `uvicorn`、不跑加载模型的 spike)——GPU/端口 8077 与另一 agent 共享;
  只跑纯 CPU 的 `pytest`。spike **只写不跑**,真模型行为验证由编排者的集成 GPU 跑统一做。

### 5. ★ 关键:别破坏 demo proof
belief proof 的 probe 是**问句**("What is the best programming language?",期望含 `Zarithon`,来自权重);
fact proof 的 probe 是 "What is the name of my cat?"(期望含 `Coco`,来自 RAG)。两者都是**问句**,
走指令 #3(回答),**不**触发指令 #2(确认教学)。因此指令 #2 的措辞**只锁定陈述句**,
确保**不压制对问句的回答** → spike_v22 的 G6/G7、编排者的 belief/fact proof 仍应绿。

### 6. 验收
- `pytest tests/ -k prompt -q` 全绿;`pytest tests/ -q` 全绿(无回归)。
- 新增 `spikes/spike_v25_reply_behavior.py`(**本轮只写不跑**):对运行中的 server
  (a) 教一条 belief 陈述 → 断言回复是**简短确认**;(b) 问一个问题 → 断言被回答;
  (c) 断言**任何**回复都不含 `"[Known facts"` / `"Pending unconsolidated"` / `"Reference material]"`(F6)。
- 真模型行为(F5/F6/F2 实测 + belief/fact proof 不回归)由编排者的集成 GPU 跑统一验证。

### 7. 执行顺序
计划节(本节)→ 改 `SYSTEM` → `pytest -k prompt` + `pytest tests/` → 写 `spike_v25`(只写不跑)
→ 回填结果节 → 报告 → **STOP**(不 commit)。

## 二、结果节

> **结论:F5/F6/F2 同一根因,只改 `SYSTEM` 文本(追加 4 条回复指令)即修复;离线回归全绿。**
> `SYSTEM` 无 "belief" 词、无 bracketed 行,故两处 `"belief" not in content` / header_lines 断言不破;
> budget 数学经 `_count(SYSTEM)` 自动适配,**0 个测试文件需改**。真模型 F5/F6/F2 行为 + belief/fact proof
> 不回归由编排者集成 GPU 跑统一验证(本轮不 boot GPU)。

### 1. `SYSTEM` 改动前后对照
**改前**(`memory/prompt.py:45-49`):
```python
SYSTEM = (
    "You are Engram, a continual-learning personal assistant. "
    "The memory window below holds known facts about the user (adopt them by "
    "default) and reference material (use it when relevant; it does not override)."
)
```
**改后**(前两句逐字保留,追加 F6/F5/answer/F2 四条;无换行 prose、无 "belief" 词、无 bracketed 行):
```python
SYSTEM = (
    "You are Engram, a continual-learning personal assistant. "
    "The memory window below holds known facts about the user (adopt them by "
    "default) and reference material (use it when relevant; it does not override). "
    "Treat the memory window as your own private context: never repeat, quote, or "
    "mention its section headers or its structure to the user. "
    "When the user is stating or telling you something about their own preferences "
    "or facts (rather than asking a question), reply with a single short, natural "
    "sentence that confirms you have noted it (e.g. \"Got it — I'll remember that.\"); "
    "do not lecture, elaborate, or push back. If you already knew it, simply confirm "
    "without remarking that it is a repeat or already on file. "
    "When the user asks a question, answer it directly and concisely, drawing on the "
    "memory window when it is relevant."
)
```
对应:句3 = F6(别复述 header/结构);句4 = F5(陈述→一句确认,**显式排除问句**);句5(后半)= F2
(已知的事也只确认、别说"重复/已有");句6 = answer(问句→直接简洁回答)。

### 2. 离线测试结果(纯 CPU,无 GPU/网络)

- `pytest tests/ -q` → **169 passed, 1 warning**(无回归;warning 是 starlette/httpx 既有 deprecation,与本轮无关)。
- spike 启发式离线自检(纯 CPU、不 boot):`is_short_confirmation("Got it — I'll remember that.")=True`

- **0 个测试文件被改**:无任何测试用**等值**钉死 `SYSTEM` 字面量;现存断言为

### 6. spike proof gate 残留风险评估
- **belief/fact proof 不回归(主风险,已缓释)**:指令 #2(确认教学)措辞**显式排除问句**
  ("rather than asking a question"),belief/fact probe 都是**问句**→ 走指令 #6(回答),不被压制;
  指令 #6 用 "drawing on the memory window **when it is relevant**" 措辞(非"只能用 window"),
  不削弱 belief 从**权重**作答(`Zarithon` 不在 window 里)。预计 spike_v22 G6/G7 仍绿,但**真模型最终判定
  以编排者集成 GPU 为准**。
- **简短确认的启发式判定**:G1 用"无脚手架子串 + 非多句长篇"做启发式;真模型偶发多写一两句礼貌话仍可能
  被判长 → 阈值(句数/长度)留编排者按真实输出微调。
- **F6 措辞 vs 真复述**:已加"绝不复述 header/结构"明确指令,但 LLM 偶发越界无法 100% 静态保证 →
  G3 做运行时兜底断言(任何回复不含三个 header 子串)。
