# v1.2 — Eval Harness (bottleneck-localization matrix)

> 计划文档(开发前)。配套规格见 `docs/eval/eval_harness.md` 与数据 schema `eval/SCHEMA.md`。
> 完工结果文档将另写 `docs/v1.2-eval-harness-results.md`。

## 目标 / 背景
`eval/` 下 `dataset.py / metrics.py / conditions.py / run_matrix.py` 是空壳,且按一套与真实
A/B/C 数据**对不上**的旧 per-item 探针模型(efficacy/paraphrase/application/locality + 从
`third_party` 加载)写的 → 全部**作废重写**(INV-E2)。

侦察已确认数据本体干净:500 条(A=370/B=60/C=70),硬不变量全过,直接用。

本 harness 的唯一目的:**定位瓶颈 module**。做一条消融阶梯,**相邻级的分数落差**把损耗归因到
某一个 module(query-split / dedup / extractor)。看的是落差,不是绝对分。

范围:只动 `eval/`(+ 新增 `tests/test_eval_*.py`)。**不改** `samples.json / memory / serving /
editing.py / generate.py`。评判模型 = Qwen(`memory.llm.complete`,默认 `qwen/qwen3.7-max`,
temperature=0),与被测 Llama-3.1-8B 天然隔离(INV-E3)。

## ★ 对规格的一处关键纠正(已对源码核实)
规格(§3/§9)说用 `reload_base` 在级间清空权重。**`reload_base` 不存在**;而且
`editing.edit` 返回的 `reset` 回调(`editor.reset_layer`)在**首条之后的序列编辑里是 no-op**
(`third_party/horen/src/models/horen/editor.py:44-56,75-77`:HoReN 只在槽位是裸 `nn.Linear`
时深拷贝 `original_layer`;第 2..N 条编辑槽位已是 `HopfieldAdapter`),且 `run_pass` 根本不
暴露它。

**解决(对应规格 open-point #1):** 统一、廉价的清空原语是
`model_host.swap_edit_module(None)`(`serving/model_host.py:69`)——它恢复 `load_base` 时**只抓
一次**的裸 `nn.Linear`(`model_host.py:59`,`_S["original"]`,全程不被改),于是下一次
`editing.edit` 解包回 base HF(`editing.py:59`)并**从 size 1 重建一个干净 codebook**。单条/序列
两种都适用,开销微秒级。`load_base()`(~90s)只在 boot 时调一次,作为断言失败时的兜底。
**规则**:每次都把 `model_host.current_model()` **新鲜取**传进 `editing.edit`/`generate`,绝不跨
reset 缓存 model 句柄。

## 锁定的决定
- **Reset**:每个 sample/bundle 的 setup 处 `swap_edit_module(None)` + `store.reset()` +
  `rag_store.reset()`;`load_base()` boot 一次(断言失败兜底)。
- **RAG k = 5**,写死并记录(= 现有默认;m=15 时把 RAG recall 卡在 5/15≈0.33,正是 recall-vs-m
  的论点)。
- **C 纳入为一个 condition**(用户决定):对 C 跑 `base/rag/edit`;Qwen 抽取所选 item name,与
  `gold_answer` 二元比对。模型只看 `name`+`blurb`(**不给 `attributes`**,那是评分 ground-truth)。
- **新增 P3_B 干净归因级**(用户决定):直接 `editing.edit` 同一个 15 条 bundle(绕过
  buffer/run_pass/dedup)+ RAG → `dedup_loss_clean = P3_B − P3.5`。规格的 `P3 − P3.5` 也保留(标注
  混入了 data-family 混淆)。
- **token 轴用 HF Llama tokenizer**,不是 tiktoken(未装、词表也不对)。`metrics.py` 收一个**注入**
  的 `count_tokens` callable(保持 GPU-free 可测);`run_matrix` 注入
  `lambda t: len(model_host.tokenizer().encode(t, add_special_tokens=False))`。
- **编辑用 `edit_prompt` 原文**(保留 cloze `___`,spike 已验形态)。`stem_of()`(去 `___`)**只**
  用于 extractor 探针的 ground-truth。
- **supersede dedup 探针 = "untested(无数据)"**(默认 §5 方案 A)。

## 各级 / 条件(实际调用)
| 级 | 数据 | Setup | 查询 / 指标 |
|---|---|---|---|
| **P1 edit-only** | A(5×zero_prior_Y) | `clean_all`;`do_edit({prompt:edit_prompt,target_new,subject})`;断言 cb==3 | cloze `edit_prompt`(efficacy)+ 每条 `queries[].q`(paraphrase)+ 一条无关 A query(locality);`gen(with_rag=False)`;locality 记 `live_score` |
| **P2 rag-only** | A | `clean_all`;`add_rag(rag_doc)`(不编辑) | `queries[].q`;`gen(with_rag=True,k=5)`;不测 locality(INV-E7) |
| **P3 edit+rag** | A | `clean_all`;`do_edit`;断言 cb==3;`add_rag(rag_doc)` | `queries[].q`;`gen(with_rag=True,k=5)` |
| **P3_B edit+rag(bundle,无 dedup)** | B(1×m=15) | `clean_all`;直接 `do_edit` 15 条;`add_rag` 每条 `rag_docs[i]` | `generation_prompt`;`gen(...,max_new_tokens=256)`;`score_B`(match_any) |
| **P3.5 序列·旁路 extractor** | B | `clean_all`;`buffer.append(MemoryItem(route=edit,status=buffer,provenance={PROV_EDIT:{stem:edit_prompt,target,subject}}))`×15;`add_rag`×15;`triggers.manual()`(记 n_written) | `generation_prompt`;同 P3_B |
| **P4 序列·真 extractor** | B | `clean_all`;`ingest([{user, rag_docs[i]}])`×15(真 extract→route→buffer/rag);`triggers.manual()` | `generation_prompt`;同上 |
| **C-base / C-rag / C-edit** | C(5) | `clean_all`;(rag:`add_rag(user_fact.rag_doc)`)/(edit:`do_edit(user_fact.edit_prompt,target_new)`) | 渲染 `question` + 15×`name: blurb`(无 attributes);`gen(with_rag=(cond==rag))`;`score_C` vs `gold_answer` |

## 落差归因 + 输出(`run_matrix` 运行时落盘)
- `query_split_loss = max(P1_eff, P2_eff) − P3_eff`(A,仅 zero_prior)。
- `dedup_loss = P3 − P3.5`(规格,标注混淆)**且** `dedup_loss_clean = P3_B − P3.5`(干净)。
- `extractor_loss = P3.5 − P4`。`bottleneck = argmax(query_split, dedup_clean, extractor)`。
- `recall_vs_m`:editing(P3_B)vs rag-only 在 m∈{5,8,11,15}(放大阶段)。
- C 各条件准确率(base/rag/edit)——隐式触发故事。
- locality(P1)误触发率;每 cell prompt tokens(含/不含 RAG 窗口)。
- 落盘:`results.json`(cells)、`summary.json`(汇总+落差表+k+threshold)、`bottleneck.md`、
  `probes.json`,放大后 `recall_vs_m.csv`。

## 先验的风险(前置 spike)
- **R1 reset 可靠性(命脉)**:见上,断言 `codebook_size==3` + base 回退;兜底 `load_base()` 每样本。
- **R2 paraphrase 检索**:编辑后模型能否答 `queries[].q`(改写句)而非只答 cloze stem?若 ≈0,把
  cloze efficacy 记为已证下界,queries 当 generalization。
- **R3 B 自由生成召回**:编辑条件在 `generation_prompt` 自我介绍上能否有非零召回(span 与 fact stem
  不同)?先在一个 bundle 上确认再信 B 各级。
- **R6 dedup "same" 探针**:问句 vs 陈述句可能判 "new";记录裁决,不强断言。

## 与规格的有意偏离
1. `swap_edit_module(None)` 取代不存在的 `reload_base` / no-op 的 `reset_layer`。
2. token 轴用 HF Llama tokenizer(注入),非 tiktoken。
3. 新增 P3_B 级 + `dedup_loss_clean`(用户批准)去混淆。
4. 新增 C base/rag/edit 条件(用户批准);§3 阶梯不变,C 作为额外一族。
5. k=5 写死(规格把取值下放)。
