# v0.2 — SPIKE 0:真 edit → 热替换 → RAG off 答对

> 阶段:关键纵切 / 串行 / 最高优先。开发前写计划,开发后补结果(遵 CLAUDE.md /docs 规则)。

## 目标
用真实 HoReN 对 `llama-3.1-8B-instruct` 做**一次**真 edit,当作可热替换的 edit module 装进 `model_host`,
在 **RAG off** 下让模型答对 —— 证明知识进了模型(adapter),不靠 prompt。顺带敲定 TBD:**产出格式** + **单条 edit 延迟**。

## 关键设计(基于精读 HoReN 源码)
- **产出格式 = 子模块(side-module)**:HoReN 把 `model.model.layers[29].mlp.down_proj`(`nn.Linear`)替换成
  `HopfieldAdapter`(codebook:`keys` buffer / `values` Parameter / `key_labels` list),base 权重冻结。
  **"edit module" = 这个 adapter**;热替换 = 对 down_proj 做 `setattr`(adapter ↔ 原始 Linear,共享同一份冻结权重)。
- **驱动用 `apply_horen_to_model(model, tok, [request], hparams)`**(`src/models/horen/horen_main.py`):
  它创建 `HOREN` wrapper、训练 adapter(Adam,`n_iter=50`,`edit_lr=1.0`)、**不 reset**(edit 默认留存),
  返回 `(HOREN_wrapper, reset_fn)`。比 `BaseEditor.edit(sequential_edit=True)` 更直接、无 pre/post-eval 干扰。
- **生成必须走 `HOREN_wrapper.generate(...)`**:它会先 `key_id = input_ids.shape[1]-1`(最后 prompt token),
  adapter 检索才对位;且 **`use_cache=False`**、`do_sample=False`(贪心),镜像官方 `test_prediction_acc`。
- **怪癖**:`build_prompt` 走 chat template,layer-29 隐状态与 raw-prompt 不同 → 检索分可能 < 0.85 → edit 不触发。
  这是要刻画的点(不触发 = v0.3 头号待办,非 SPIKE 0 失败)。

## 硬编 edit(ZsRE record[0])
- prompt `"What university did Watts Humphrey attend?"`,target_new `"Illinois Institute of Technology"`,
  subject `"Watts Humphrey"`。base 原答 ≈ "Trinity College" ≠ target → 干净前后对比。

## 实现
- `serving/model_host.py`:`load_base()`(读 yaml → `HORENHyperParams` → `from_pretrained` bf16 → cuda:0;
  解析 layer-29 down_proj 的 parent/attr/原始 Linear)、`current_model()`、`swap_edit_module(m|None)`(热替换)、
  辅助 `tokenizer()/hparams()/edit_module()/register_edit_module()`。
- `editing.py`:`edit(model, memory)` 调 `apply_horen_to_model`,取出活的 `HopfieldAdapter`,计时,返回 adapter+wrapper+耗时+codebook_size。
- `generate.py`:`generate(query, *, model, ...)` 贪心解码,支持 raw-prompt 与 `build_prompt`(chat)两模式。
- `memory/prompt.py`:落地 `build_prompt`(SYSTEM + RAG 窗口[buffer 段/docs 段,可空] + query)。
- `spikes/spike0_edit_swap_generate.py`:driver,硬编 fact,串 PRE → edit → POST → 热替换 ↔ → chat 路径 → locality,打印 PASS/FAIL + 延迟。

## 护栏
- 绝不装/降级 torch(已复核仍 2.8.0+cu128);只跑 N=1,sanity 出结果即停下报告;同一错误修 2 次不通即停。

## 结果(SPIKE 0 — PASS ✅,N=1 sanity)

| 指标 | 结果 |
|---|---|
| Reliability(edit 生效)| ✅ token-EM **1.0**;POST raw-prompt 答 `Illinois Institute of Technology`,PRE 答 `Watts Humphrey attended`(≠ target)|
| Generalization(改写句)| ✅ rephrase 命中 target(此条恰过 0.85 阈值;N=1 不代表整体,论文里通常偏低)|
| Locality(无关输入)| ✅ 无关 prompt 仍答 `Hugo Weaving`,未误伤 |

- **热替换**:`swap_edit_module(None)` → 回退 base(`Watts Humphrey attended`);`swap_edit_module(adapter)` → 恢复正确 → **edit 活在可换 adapter 里,不在 base 权重**(零拷贝、单次 `setattr`)。
- **单条 edit 延迟 = 4.54 s**(n_iter=50, edit_lr=1.0);模型加载 52.1 s(一次性常驻);**显存峰值 16.2 GB / 46 GB**;**无报错**(退出码 0,仅 timm/`torch_dtype` 无害 warning)。
- **产出格式确认 = `HopfieldAdapter` 子模块** @ `layers[29].mlp.down_proj`(codebook keys/values/key_labels,base 冻结)。**不是** state_dict / delta。

### ⚠️ 头号发现 → v0.3
我们的 **chat 推理路径(`build_prompt` + chat template,RAG off)edit 不触发**:答
`I'm not aware of any information about a person named Watts Humphrey...`。
原因:HoReN 检索 key 建在 **raw-prompt** 的 layer-29 隐状态;chat template 包裹后隐状态偏移 → 检索分 < 0.85 → 不注入。
v0.3 候选解:edit 时用 chat 模板化 prompt 建 key / 推理时给检索单独喂 raw query / 调阈值或 query 选择策略。
raw-prompt 路径已完全打通,底座稳。
