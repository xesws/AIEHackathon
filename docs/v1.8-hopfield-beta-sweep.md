# v1.8 — Hopfield retrieval β 全局扫描:能否消 sibling 互串而不塌 PARA(measurement-only)

> 依据: v1.7 结论(`docs/v1.7-locality-neg-selection-ablation.md`)收窄的真问题 = **同主语 sibling 互串** + 本轮用户指令
> 类型: **measurement-only** · 只扫 `hopfield_retrieval_beta` **一个**变量 · 独立 model 副本 · toggle/default OFF · 不上线
> 目标: 判断锐化 Hopfield retrieval(调高 β)能否让 read-key 只认最近的 stored key,从而消 sibling 互串,且**不**塌 PARA。

## 背景锚点(别再质疑)
- v1.7:触目的 fact NEGxfire 11/20 主要是 **sibling 同主语近重复** artifact;系统对**无关世界知识**不污染(generic-NEG 0/40)。真问题收窄为 **同一 JQ 主语下多条相似 personal fact 之间 deferral 互串**。见 [[horen-key-cone-collapse]]。
- 候选解之一:**锐化 Hopfield retrieval** —— 调高 β 让 read-key 只认最近的 stored key,不被 sibling 的 stored key 拉过去。当前 yaml `hopfield_retrieval_beta=20`。
- 本轮**只做【全局】扫 β**。**明确不做**动态 / per-query β —— 缺一个干净的"该调多少"信号,且破坏可复现性。

## 关键机理(决定 β 能做什么 — 跑前预判,跑后核对)
生产 deferral gate = `keying.score` → `adapter._query`,codebook = `[random_placeholder, write_key]`(2 键,一随机一真),yaml `max_iter=1 / alpha=0.1`:
- `scores = β·[q·placeholder, q·write]`;placeholder 随机近正交 → `q·placeholder ≈ 0`,记 `w = q·write`(read↔write 原始 cosine)。
- write 的 softmax 权重 = `sigmoid(β·w)`。**β=20、w≈0.87 → sigmoid(17.4) ≈ 1.0 = 已饱和** → `retrieved ≈ write`。
- `qt = normalize(0.9·q + 0.1·retrieved)`;最终 score = `qt·write`,与 0.85 比。
- ⇒ **预判**:β∈{20,40,80,160,320} 全落在饱和区,gate 分数对 β **不敏感(表应近乎持平)**;β 唯一能动分数的区间是 β≲10(softmax 没饱和,分数滑向原始 cosine)。本轮**测**,不假设;给 β=20 整完整性闸把关。

## 实验(复用 S3 sibling-NEG harness:同 10 真样本 / 同 seed / 同 `keying.score` / 阈值 0.85,全部不变;**只扫 β**)
- β ∈ **{20(现状/baseline), 40, 80, 160, 320}**(任务指定集 = headline)。
- 诊断扩展(同一变量、更密采样,用于裁决"有没有干净窗口 / 是否一起动"):β ∈ {1, 2, 5, 10}(< 20,落在非饱和区,显示分数如何滑动)。
- 10 真样本 by id:facts A0001–A0005,beliefs A0139–A0143(同 v1.5/v1.7 的 S3 arm)。
- key = S3 live chat key(`compute_key(templated=True)`);**key 与 β 无关**(compute_key 不调 `_query`)→ keys 算一次,每个 β 仅重打分。**write/read 用同一 β**(同一 `_query`,同一 placeholder)。
- 每个 β 报告(fact / belief 分组):`DIRECT均值 | PARA均值 | NEG地板 | PARAfire(/5) | NEGxfire(/20)`。NEG = 同组 sibling 的 DIRECT(5×4=20)。

## ★ 裁决(anti-false-fix,同前几轮)
寻找是否存在一个 β 使得 **NEGxfire 显著↓(sibling 不串)** **同时** **PARAfire 保持高(≈5/5)**。
- 存在 → β 是解,报告该值。
- NEGxfire 与 PARAfire **总一起动**(β 高两个都不触发=整体拒识;β 低两个都触发)→ **没有干净窗口** → β 救不了 → cat/snack 的 key 距离太近、retrieval 锐度解决不了表示重叠。
裁决 = (NEGxfire↓ ∧ PARAfire 保持) 的**联合**,不是单看 NEGxfire。

### 机理(为什么 β 几乎无效)
gate = `keying.score` → `_query`,codebook=`[random_placeholder, write]`,`max_iter=1 / alpha=0.1`:
`score = normalize(0.9·read + 0.1·retrieved)·write`,其中 `retrieved=normalize(softmax(β·[≈0, w])@K)`,`w=read·write`。
- Hopfield 项**只占 10% 权重**(alpha=0.1、单步),且对 `w≳0.2`、`β≥~5` 时 softmax 已饱和(`retrieved≈write`)→ **score ≈ raw cosine `w` 的一个固定单调函数,在 β∈[1,320] 上几乎不随 β 变**。
- ⇒ 调高 β:no-op(表逐位持平)。调低 β(→1):整体 score 微微滑向原始 cosine(DIRECT/PARA/NEGfloor **一起**降 ~0.002),没有"只压 NEG 不压 PARA"。

### 裁决(joint anti-false-fix:NEGxfire↓ ∧ PARAfire 保持)
- **fact(真问题所在组):没有任何干净窗口** —— NEGxfire 在 β=1..320 **每一点都是 11/20**,PARAfire 恒 5/5。β 对 fact 是**完全无效**。
- **belief:harness 自动标了一个名义"窗口"β=1(NEGxfire 7→6 且 PARAfire 仍 5/5)。诚实判读 = 这不是 fix**:
  1. 只是 **1 个 NEG 对**从 ~0.85 擦边掉到刚好 < 阈值(NEGfloor 0.871→0.868),不是"显著下降";
  2. 方向是**调钝(β↓)不是调锐(β↑)**——与"锐化消互串"的假设相反;
  3. 伴随 DIRECT/PARA/NEGfloor **一起**下滑(co-movement),正是"β 低了大家一起降"那支;
  4. **fact 组该 β 纹丝不动(11/20)**——真正有 sibling 互串问题的组完全不动。
- ⇒ **全 β 轴无干净窗口。** {20..320} 区间 β 已饱和、动都不动;< 20 时唯一效应是 PARA 与 sibling-NEG **一起**下滑。两支都不给"PARA 高、NEG 低"。

### 4 问回答
1. **有没有 β 同时 NEGxfire↓ + PARAfire 保持?** fact:**没有**(11/20 恒定,1→320 全平)。belief:仅 β=1 出现 7→6 的名义窗口,但那是单个擦边对 + 调钝方向 + co-movement,**不是真 fix**,且 fact 不动。**结论:没有一个 β 是解。**
2. **还是 NEG/PARA 总一起动(无干净窗口)?** 是。{20..320} β 饱和 → gate **逐位持平**(调高 β = no-op);< 20 → 所有分数(含 PARA 与 NEG)**一起**滑向 raw cosine。**全轴无干净窗口。**
3. **fact vs belief 差异:** fact **对 β 完全免疫**(11/20 处处相同)——fact src 高度同构(共享 "What is the [X] of JQ's [Y]" + 共享 JQ 主语),sibling NEGfloor 0.898 更高/更近;belief 主题各异(NEGfloor 0.871)、有一个 NEG 对擦边 0.85,故仅它在 β=1 闪一下。**即:越难的 fact 组,β 越救不动。**
4. **一句话:β 不是 sibling 互串的解,是 key 太近、retrieval 锐度救不了。** `max_iter=1/alpha=0.1` 下 gate 分 ≈ raw read↔write cosine 的固定单调函数(Hopfield 项 ≤10% 且 β≈5 即饱和),调高 β 逐位 no-op、调低 β 只让 PARA 与 sibling-NEG **一起**下滑;cat/snack 的 sibling key cosine 本就交织(sibling-NEG 高至 0.898 ≥ 部分 PARA,见 [[horen-key-cone-collapse]]),retrieval 锐度无法分离表示重叠。

### 结论 & 边界
- ✅ 证实 **β(全局,固定 max_iter=1/alpha=0.1)不能消 sibling 互串而不塌 PARA**;排除"调 β"这条捷径。
- 真问题仍是 v1.7 收窄的 **同主语 sibling 表示重叠**;要解需更强 key 表示(换 layer / scaffold-free 池化 / learned 投影),非检索锐度。
- 边界:本轮固定 `max_iter=1`(yaml)。理论上 β 要真正"多步锐化拉向最近键"需 `max_iter>1`,但那会**同时**动 max_iter,超出"只扫 β"且改可复现性——本轮明确不做;如要可另开一轮(measurement-only)单独评估 `max_iter` 的联合效应。
- 护栏遵守:toggle/default OFF、未上线;只扫 β 一个变量;未做动态/per-query β;write/read 同 β;未碰 live server/codebook/memory/editing/samples.json/yaml;β=20 复现 S3。**STOP。**
