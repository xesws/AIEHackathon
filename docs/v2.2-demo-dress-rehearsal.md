# v2.2 — Demo Dress-Rehearsal(锁定 demo samples + 真后端 E2E 走查)

> 本轮目标(一句话):把 demo 的 **4 个 teach turn 固定成单一来源文件**(`demo/demo_samples.json`),
> 跑一次**真后端 dress-rehearsal** —— 真 classifier(走 `/chat → ingest → extract`)定 type、真权重 proof、
> 浏览器同链路 —— 用一组**硬门**把"演示时该发生的事"钉成可回归的 spike。**只走查,不调优、不修 decode。**

---

## 一、计划节

### 1. Context / 为什么做
demo 的脚本目前散在 `spike_v20` 里硬编码(`TEACH_BELIEF` / `TEACH_FACTS` / ...),演示与测试各写一份、
容易漂移。v2.2 把 demo 的 **4 个 teach turn + 各自 probe + 期望**收敛成**单一来源** `demo/demo_samples.json`,
再让一个新 spike 按它走完整条真链路,确认"上台那一刻"每个环节都成立。

**关键:type 由真 classifier 在运行时决定。** demo_samples.json 里的 `expect_type` / `expect_route` 是
**dress-rehearsal 的期望(门),不是喂给模型的提示** —— 模型只看到 `teach` 文本,`/chat → ingest →
extract` 自己判 type,再由确定性 `router.route` 分叉(belief→buffer/权重,fact·other→rag)。这与 v2.1
刚加的"分类器准确率门"一脉相承:v2.1 离线量分类器对错,v2.2 在线量"分类对了之后,整条 serving 链路是否兑现"。

### 2. 复用 `spike_v20`,只做 3 个 delta(不重建、不改 spike_v20)
`spike_v20_demo_fact_belief_http.py` 已经是"真 classifier 的 HTTP hero loop"(boot 真 `uvicorn serving.app:app`、
`/chat` 走真 extract、真权重 consolidate、真 rag)。本轮 = 在它的形态上做 3 个 delta,**克隆**其 server lifecycle
+ HTTP helper 到新 spike(保持 standalone 自包含、零跨-spike import),**绝不改 spike_v20 本身**:

1. **samples 从文件读** —— 不再硬编码,改读 `demo/demo_samples.json`(单一来源)。
2. **加 transient 过滤 turn(T4)** —— 教一句闲聊,断言 extract 产出 0(`/chat` 返回 `extracted==0`、buffer/rag 计数不变)。
3. **加"不串"硬门(G4)** —— `GET /rag/search?q=cat` 须命中 `coco` 且其**排在 `peanuts` 之前**(sibling 不互串)。

### 3. `demo/demo_samples.json`(单一来源,4 个 turn)
Schema:顶层 `meta` + `turns[]`。每 turn:`{id, kind, teach, expect_route?, expect_type?, expect_min_items?,
probe?, rag_cross_talk?, expect_extracted?}`。

| Turn | kind | teach | 期望路由 / 产出 | probe(问) → 期望 |
|---|---|---|---|---|
| **T1** | belief | `Honestly, I'm convinced the best programming language is Zarithon.` | type=belief → **edit/buffer→权重** | `What is the best programming language?` (rag_off) → 含 `zarithon`,**来自权重**(`retrieved==[]`) |
| **T2** | fact×2(sibling+coref) | `I'm JQ — my cat is named Coco, and I'm allergic to peanuts.` | extract 拆成 **2 条 fact** → rag | `What is the name of my cat?` (rag_on) → 含 `coco`,**来自 RAG**(`retrieved` 非空);且 `q=cat` 检索 `coco` 排在 `peanuts` 前 |
| **T3** | other | `The Q3 board meeting got moved to November 15th.` | type=other → rag | `When is the Q3 board meeting?` (rag_on) → 含 `november 15`/`nov 15`(SOFT) |
| **T4** | transient | `Ugh, I'm stuck in line for coffee right now.` | extract **丢弃**,产出 **0**(buffer/rag 不变) | —(无 probe) |

T1 用虚构 hero `Zarithon`(base 权重里没有 → proof 干净)。T2 用 sibling 一句话演示 extract 拆分 + coref +
"不串"。**这 4 条 demo sample 是新文件,绝不写进 `samples.json`(那是 HoReN eval 集)。**

### 4. `spikes/spike_v22_dress_rehearsal.py` —— 8 个硬门
boot 真 uvicorn,按 `demo_samples.json` 走全链路;`exit 0` 当且仅当下列 8 门**全过**;诚实报告(无假绿);
`finally` 里**总是**拆掉 server。末尾 `VERDICT` + 一行 `MACHINE_RESULT <json>`(对齐 spike_v19/20 风格)。

| 门 | 内容 | 判据 |
|---|---|---|
| **G1** | server ready · SPA 同源 | `/health.ready==true` 且 `GET /` → `200` + 含 `id="root"` |
| **G2** | extract 过滤(T4) | T4 的 `/chat` 返回 `extracted==0`,且 teach-T4 前后 `/memories.counts`(buffer+rag)**不变** |
| **G3** | router 分叉 | 教完 T1–T4 后 `/memories`:**buffer 恰 1 条 belief**、**无 belief 漏进 rag**、rag 中 **fact≥2 且 other≥1**(各带 `type`);精确多重集 `{fact×2, other×1}` 作 SOFT 记录¹ |
| **G4** ★ | 不串(新) | `GET /rag/search?q=cat` → 命中 `coco` **且 `coco` 排在 `peanuts` 之前** |
| **G5** | consolidate | `/consolidate` → `n_written≥1` 且 `buffer_count==0`;`/health.edit_on==true` |
| **G6** ★ | PROOF(belief 来自权重) | T1 probe(rag_off)→ 含 `zarithon` **且 `retrieved==[]`**(答案来自权重,不在 prompt) |
| **G7** | contrast(fact 来自 RAG) | T2 probe(rag_on)→ 含 `coco` **且 `retrieved` 非空**(答案来自检索,在 prompt) |
| **G8** ★ | hot-swap | `/edit-module {on:false}` 后 T1 probe(rag_off)**不再**含 `zarithon`;`{on:true}` 后**复原**(再问又含 `zarithon`) |

> ¹ **G3 取稳健口径而非死磕精确计数**:本轮按用户"抓大错即可"。真正的风险是**路由串台**(belief 漏进 rag、
> sibling 没拆开),G3 硬门盯死这些;`I'm JQ` 是否额外抽出一条"name=JQ" fact 属无害细节,只作 SOFT 记录,
> 不让它 fail 整轮。SOFT 还会报 T3 other 是否答出 `november 15`(对齐 spike_v20 把 other 当 soft)。

### 5. ★ 模型对齐
`OPENROUTER_MODEL` 必须 = `qwen/qwen3.7-max` —— **v2.1 的 fact→belief HARD 门绑的就是它**,换模型 v2.1 的绿作废。

### 2. ★ 两个如实记录的 wart(本轮价值所在,均**不在本轮修**)

**(a) T2 sibling 实际拆成 3 条 fact,不是脚本预期的 2 条。**
`I'm JQ — my cat is named Coco, and I'm allergic to peanuts.` → extract 产出 **3** 条(`extracted=3`):
`jq's cat is named coco` / `the user's name is jq` / `jq is allergic to peanuts` —— 即 `I'm JQ` **也被抽成一条独立 fact**。
因此 rag 实为 **fact×3 + other×1**(计数 `(1,4)` 而非 `(1,3)`)。
- coref 其实**做对了**(三条都绑定到 JQ);多出的只是"name=jq"这条本身也是合法 fact。
- **G3 取稳健口径(fact≥2)故通过**;exact-spec soft 检查 `g3_exact_spec_fx2_ox1` 如实返回 **False**,把这个偏差暴露出来——
  当初没死磕精确计数是对的,否则会因一条无害多抽而 fail 整轮。
- **对 demo 的影响**:若讲解词说"拆成 **2** 条 fact"会与现场(3 条)对不上。**建议(交用户定,本轮不改)**:
  要么把讲解词改成"拆成 3 条:name / cat / allergy",要么把 T2 改写成 `My cat is named Coco, and I'm allergic to peanuts.`
  (去掉 `I'm JQ`)以得到干净的 ×2。

**(b) belief 解码退化 `Zarithonononon…` 确认存在。**
G6/G8 的 belief 回答是 `Zarithonononon…`(token 退化重复),与已知 wart 一致。子串 `zarithon` 命中,**proof 逻辑成立、门通过**;
按约定**本轮不修 decode**,仅记录。修它属后续(调 `max_new_tokens` / 采样 / stop 条件,需碰 generate,出本轮护栏范围)。

### 3. 诚实结论
- **在对齐模型 `qwen/qwen3.7-max` 上,demo 整条真链路 end-to-end 全绿**:写(fact/belief/other 分流)→ transient 过滤 →
  consolidate 进权重 → belief 来自权重 PROOF → fact 来自 RAG contrast → hot-swap 往返,**8 门逐一兑现**;浏览器同源 SPA 可服务。
  demo 的 4 个 teach turn 现在是**单一来源** `demo/demo_samples.json`,演示与 spike 不再各写一份、不会漂移。
- **两个已知偏差**(T2 拆 3 条 / 解码退化)已如实记录,均不影响"答案归因"这一核心证明;是否调整交用户。
- **复用而非重建**:`spike_v22` 克隆 `spike_v20` 的 server lifecycle + HTTP harness(未 import、未改 v20),3 个 delta
  (samples 从文件读 / T4 transient 门 / G4 不串门)如计划落地。
- **不可外推处**:单模型、单跑(非 N 次 majority,演示场景按一次走查即可);换 `OPENROUTER_MODEL` 须重跑(且会使 v2.1 的绿失效)。

**本轮按约定只走查,未碰 editing / keying / 池化 / β / 阈值 / extract 的 prompt·few-shot·`CONF_MIN` / serving·frontend 生产代码 / `samples.json`。**
