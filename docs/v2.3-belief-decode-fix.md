> 本轮目标(一句话):只动 `/chat` 对话路径的 decode 参数,把强编辑命中下 belief 回答的
> token 死循环(`Zarithonononon…`)修成干净的 `Zarithon`,**proof 逻辑与 eval greedy 语义全部不动**。

## 一、计划节

### 1. 根因(已定,不再探)
belief 回答退化成 `Zarithonononon…` 是 **强编辑命中 + greedy(`do_sample=False`)+ `use_cache=False`** 三者叠加导致的
**token 死循环**,与 target 是哪个词**无关**。HopfieldAdapter 命中后把"Zarithon"对应方向的 logits 持续顶高,greedy 每步都取它、
`use_cache=False` 又让每步重算全程,于是 `on` 这个 subword 被无限重复。

→ **不换 target**:`Zarithon` 保持不动(虚构词,已被 `spike_v19`/`spike_v22` 验证,base 里没有 → proof baseline≈0、
归因干净)。**纯 decode 参数修。**

### 2. 改动(最小面)
给 `generate.generate()` 加两个 **默认关闭** 的 decode kwarg,透传到 `_greedy` 的 `model.generate(...)`;
**只在 `serving/app.py` 的 `/chat` 调用点打开**:

| 参数 | 默认(off) | `/chat` 启用值 | 作用 |
|---|---|---|---|
| `no_repeat_ngram_size` | `0`(HF 默认) | `3` | 禁止任意 3-gram 重复 → 直接掐断 `on on on…` |
| `repetition_penalty` | `1.0`(HF 默认) | `1.3` | 对已生成 token 的 logits 降权 → 退出循环倾向 |

- `do_sample` **保持 `False`**(不引入采样,proof 要确定性);两参数都是 **logits processor**,在 greedy 下照常生效、不破坏确定性。
- **`_greedy` 默认行为零改动**:仅当参数 != HF 默认(`0` / `1.0`)时才把它注入 `model.generate(**extra)`;off 时 `extra={}`,
  调用与现状**字节一致** → `spike_v19`/`spike_v22` proof 路径 + HoReN eval-parity 的 greedy 语义不变。
- **透传链已验证安全**:HoReN wrapper(`third_party/horen/src/models/horen/editor.py:67`)`generate(self, *args, **kwargs)`
  设完 `key_id` 后 `return self.model.generate(*args, **kwargs)` —— **全量转发** kwargs 给底层 HF generate,
  故新参数能到 HF,**无需碰 wrapper**。
