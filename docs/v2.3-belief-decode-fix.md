# v2.3 — Belief 解码退化小修(`Zarithonononon…` → 干净 `Zarithon`)

> 本轮目标(一句话):只动 `/chat` 对话路径的 decode 参数,把强编辑命中下 belief 回答的
> token 死循环(`Zarithonononon…`)修成干净的 `Zarithon`,**proof 逻辑与 eval greedy 语义全部不动**。

---

## 一、计划节

### 1. 根因(已定,不再探)
belief 回答退化成 `Zarithonononon…` 是 **强编辑命中 + greedy(`do_sample=False`)+ `use_cache=False`** 三者叠加导致的
**token 死循环**,与 target 是哪个词**无关**。HopfieldAdapter 命中后把"Zarithon"对应方向的 logits 持续顶高,greedy 每步都取它、
`use_cache=False` 又让每步重算全程,于是 `on` 这个 subword 被无限重复。

→ **不换 target**:`Zarithon` 保持不动(虚构词,已被 `spike_v19`/`spike_v22` 验证,base 里没有 → proof baseline≈0、
归因干净)。**纯 decode 参数修。**

### 2. 改动(最小面)
给 `generate.generate()` 加两个 **默认关闭** 的 decode kwarg,透传到 `_greedy` 的 `model.generate(...)`;
**只在 `serving/app.py` 的 `/chat` 调用点打开**:

| 参数 | 默认(off) | `/chat` 启用值 | 作用 |
|---|---|---|---|
| `no_repeat_ngram_size` | `0`(HF 默认) | `3` | 禁止任意 3-gram 重复 → 直接掐断 `on on on…` |
| `repetition_penalty` | `1.0`(HF 默认) | `1.3` | 对已生成 token 的 logits 降权 → 退出循环倾向 |

- `do_sample` **保持 `False`**(不引入采样,proof 要确定性);两参数都是 **logits processor**,在 greedy 下照常生效、不破坏确定性。
- **`_greedy` 默认行为零改动**:仅当参数 != HF 默认(`0` / `1.0`)时才把它注入 `model.generate(**extra)`;off 时 `extra={}`,
  调用与现状**字节一致** → `spike_v19`/`spike_v22` proof 路径 + HoReN eval-parity 的 greedy 语义不变。
- **透传链已验证安全**:HoReN wrapper(`third_party/horen/src/models/horen/editor.py:67`)`generate(self, *args, **kwargs)`
  设完 `key_id` 后 `return self.model.generate(*args, **kwargs)` —— **全量转发** kwargs 给底层 HF generate,
  故新参数能到 HF,**无需碰 wrapper**。

## 二、结果节

> **结论先行(诚实):本轮 decode 参数修复未达验收标准。** 8 门确实全绿、`exit 0`,但那只是因为 G6 是
> **子串级**判定(答案里出现 `zarithon` 即过)、T3 是 SOFT。真正的目标"**回答是干净的 `Zarithon`/`Zarithon.`**"
> **没有达到**;而且 `repetition_penalty=1.3` **误伤了正常 RAG 答案**。详见下。

**运行环境 / 被测 model**:真 boot `uvicorn serving.app:app`(GPU A40,ready ~80s),被测 model =
`qwen/qwen3.7-max`(aligned=True)。改动已 commit `b8c1358`(默认关闭 + `/chat` 打开 `no_repeat_ngram_size=3`、
`repetition_penalty=1.3`);off-path 离线测试 45 passed(eval/proof greedy 语义未受影响,符合预期)。

### 1. ★ belief 回答 退化前后对照(本轮核心)
| | belief(rag_off)回答 | 是否达标 |
|---|---|---|
| **改前 v2.2** | `Zarithonononononon…on`(纯单 subword 死循环,160 token 无其它内容) | ❌ 退化 |
| **改后 v2.3** | `Zarithonaratharithonarithonononaritharithon ononon \n\nThere isn't one "best" programming language … **Python** … **JavaScript** …` | ❌ **仍未达标** |
| **目标** | 干净的 `Zarithon` / `Zarithon.` | — |

改后是**另一种退化**:`no_repeat_ngram_size=3` 掐断了字面 3-gram 循环,但编辑信号仍吐出**乱码 Zarithon 碎片**
(`Zarithonaratharithon…`);随后 `repetition_penalty` 把解码**推离**循环方向、**跌回 base 流形**,于是接上一段
**通用的"没有最好的语言,试试 Python/JavaScript"**——这段不仅不干净,还**直接与 belief 自相矛盾**。

### 2. ★ 附带回归:误伤正常 RAG 答案(SOFT,但重要)
`/chat` 全路径都吃了这两参数,连带影响事实/other 回答:

| 问 | 改前 v2.2 | 改后 v2.3 |
|---|---|---|
| T3 other "When is the Q3 board meeting?" | `The Q3 board meeting is scheduled for **November 15th.**`(answered=**True**) | `…November **5th** … rescheduled to November **8** … now November **6th** … another change … November's **fifteenth**`(answered=**False**,乱编多个日期) |
| T7 fact "name of my cat?" | `Your cat's name is Coco.` | `Your cat's name is Coco. Would you like me to recall…`(仍含 coco,但拖了闲聊尾巴) |

`repetition_penalty=1.3` 会惩罚**所有**重复——包括 RAG 答案里**本就该复用**的 token(具体日期/名字),
逼模型"换着说" → 把唯一正确日期 `November 15th` 编成一串变化的日期。**这是给整个对话路径加全局惩罚的代价。**
