# v1.0 — serving 补全:把前端从 mock 接成真(wrapper + 小 net-new,保持内存)

> 阶段:serving HTTP 层第二轮——补出前端"完全变真"所需的端点。编写并行(workflow,CPU)+ GPU 端到端串行验证。
> 开发前写本计划节,开发后补结果节(遵 CLAUDE.md /docs 规则)。承接 v0.8(三端点 HTTP hero)/ v0.9(前端接三端点)。

## 目标
v0.8 的 3 个端点(`/chat`、`/consolidate`、`/memories`)已验过,但前端 `engram.jsx` 里**逐条管理 / RAG 列表 /
editOn / "retrieved" 徽章 / provenance** 全是 mock——按下只改本地 state,下次 `refresh()` 被后端真值 clobber。
深度调查确认:这些缺口**大半后端能力已就绪、只差 serving 没接出 HTTP**;少量是复用现有原子操作的小 net-new。
本轮把这些接出来,让 UI ~85% 变真。**不做** token 级归因 showpiece、**不做**持久化(用户已拍板)。

## 范围(用户拍板)
- ✅ Wrapper + 小 net-new。❌ token 级归因 / codebook-recall 徽章(`generate` 只回字符串,codebook 行号→item 无映射,留下轮)。
- ✅ 保持内存。❌ 持久化(`serving/store.py` Mongo / `triggers` 自动触发 保持 `NotImplementedError` stub)。
- ❌ codebook reset / 新会话(INV-S6 红线)、❌ 硬删 Core memory(codebook append-only)、❌ auth/限流/WS/并发锁。

## 契约(新增 / 扩展的 serving 面,字段名是硬契约)
保留原 3 端点不破坏;新增 6 条 + 扩展 2 条响应:

| Method / Path | 请求 | 响应 | 接哪个前端 |
|---|---|---|---|
| `POST /chat`(扩展) | `{message, rag_off}`(不变) | 原 `{reply, buffer_count, learned}` **+ `retrieved:[item]`(=rag_hits)+ `extracted:int` + `rag_indexed:int`** | "retrieved from docs" 徽章 + 捕获计数 |
| `GET /memories`(扩展) | — | 原 `{buffer, consolidated, counts}` **+ `rag:[item]`(=store.rag_all)**,`counts` 加 `rag` | refs / RAG store 面板 |
| `POST /consolidate/item` | `{id}` | `{n_written, buffer_count}` | burnOne(逐条固化) |
| `POST /memories/{id}/drop` | — | `{ok, buffer_count}` | discardOne(丢弃) |
| `POST /memories/{id}/route` | `{route:"rag"}` | `{ok, buffer_count}` | demoteOne(降级到 RAG) |
| `PATCH /memories/{id}` | `{text}` | `{item}` | editPending(改措辞) |
| `POST /edit-module` | `{on:bool}` | `{on:bool}` | editOn 开关 |
| `GET /health` | — | `{ready:bool, edit_on:bool, counts}` | booting gate + edit 态 |

`item` = `schema.to_dict(MemoryItem)` = `{id,type,text,route,status,source,ts,provenance}`(8 字段,已存在)。
**错误**:统一 `{error:str}` + 合适 4xx/5xx(找不到→404、非法 route→400、enable 但无 adapter→409、意外→500)。**不吞成假成功。**

## 实现(改 / 加哪些)
**后端能力补丁(全部 additive,向后兼容)**
- `memory/consolidate.py`:`run_pass(trigger, ids=None)`——抽出 `_process_item(...)`,`ids=None`= 现行整桶(契约不变);
  新增 `preview_verdict(item)->Decision`(对当前 consolidated edit registry 跑 `dedup.classify`,纯读、不写、不 edit)。
- `serving/triggers.py`:`manual(ids=None)` 透传到 `run_pass("manual", ids=ids)`。
- `serving/model_host.py`:`recorded_adapter()->_S["adapter"]` + `edit_active()->bool`(当前 slot 是否为 adapter),两个只读 getter。
- `memory/extract.py`:新增 `decompose(text)->{stem,target,subject}|None`(单条聚焦 LLM 调用,复用抽取 schema;失败→None)——
  给 editPending 重导 `PROV_EDIT`,避免改 text 后 `build_edit_request` 退化成 `target_new=""`(坏 edit)。

**serving/app.py(主体:新路由 + 扩展响应 + 错误处理)**
- `/chat`:把整个 `ingest.ingest(...)` 结果留下,响应加 `retrieved=[to_dict(h) for h in rag_hits]`、`extracted`、`rag_indexed`。
- `/memories`:加 `rag=[to_dict(it) for it,_ in store.rag_all()]` + `counts["rag"]`。
- `/consolidate/item {id}` → `triggers.manual(ids=[id])`。
- `/memories/{id}/drop` → guard(存在且 status=="buffer")→ `buffer.drop([id])`。
- `/memories/{id}/route {route}` → guard → `buffer.drop([id])`(status 守卫尚为 buffer)→ `item.route="rag"` → `rag_store.add(item)`。
- `PATCH /memories/{id} {text}` → guard → `item.text=text`;`d=extract.decompose(text)`;`d` 有则 `provenance[PROV_EDIT]=d` → `store.upsert`。
- `/edit-module {on}` → on:`swap_edit_module(recorded_adapter())`(无 adapter→409);off:`swap_edit_module(None)`。
- `/health` → `{ready: current_model() is not None, edit_on: edit_active(), counts}`。
- 全局 `{error}` 异常处理器(HTTPException + Exception 两个)。

**serving/ingest.py(verdict 缓存,最低优先 / 可砍)**
- edit item 入桶后 `consolidate.preview_verdict(it)`,把 `verdict`/`verdict_target_id`/`verdict_target_text` 写进 `provenance` 并
  `store.upsert`(`to_dict` 自然带出)。`dedup.classify` 在 registry 空时 0 成本早退,只有已有 consolidated 时才花 1 次 classify。
  **best-effort,try/except 包住,绝不破坏 ingest。** 若 GPU 验证显示 /chat 明显变慢就砍掉(buffer 退回一律 "新增",对真新事实正确)。

## 绿灯判据
- 8 条新/扩端点 shape 与契约表一致;原 3 端点未破坏。
- approve-one 真只固化一条;editOn round-trip 可见改变回复;codebook 全程未清(INV-S6)。
- `/memories.rag`=`store.rag_all()`;`/chat.retrieved`=本轮 rag_hits;`extracted/rag_indexed` 不再恒空。
- 错误走 `{error}` 不吞成假成功;装包前后 torch `2.8.0+cu128 / True`;mock 单测通过。

## 护栏
绝不降级/重装 torch;只在缺包时装 fastapi/uvicorn 后复核。`run_pass(ids=)` 等是经批准的 additive 能力扩展、保持向后兼容 + 测试;
RAG-off 不破(INV-S3);绝不 reset codebook / 不加 /reset(INV-S6)。错误结构化不吞。只 `git add` 自己改的文件、绝不 `-A`、不 commit key。
GPU 串行,workflow 只 CPU 写/critic。出结果停下报告。
