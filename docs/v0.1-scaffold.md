# v0.1 — 项目 scaffold(脚手架)

> 阶段:项目初始化 / scaffold-only。本文档遵循 CLAUDE.md:**开发前写计划,开发后补总结**。

## 目标
为 Engram(continual-learning personal agent)搭好整套架构骨架,**只建 stub,不写任何实现**:
每个代码文件 = module docstring + 函数/类签名 + TODO,函数体一律 `raise NotImplementedError`
(`schema` 的 dataclass 字段声明即接口本身,不算实现)。
HoReN 是既有外部依赖(impl of arXiv 2605.08143),**不在本仓库实现**,作为依赖 vendored 到
`third_party/horen`,绝不把 HoReN 逻辑搬进我们自己的模块。

## 范围(本阶段产出)
- 顶层:`editing.py`(HoReN 薄封装)、`generate.py`(在 edited model 上推理)、`requirements.txt`、`.gitignore`
- `memory/`(核心 IP):`schema` / `extract` / `router` / `buffer` / `rag_store` / `dedup` / `consolidate` / `prompt`
- `serving/`:`app`(FastAPI)/ `model_host`(热插拔)/ `store`(Mongo)/ `triggers`
- `eval/`:`dataset` / `conditions` / `metrics` / `run_matrix`
- `frontend/`:`index.html`(占位 demo UI)
- 依赖:`third_party/horen`(从 `xesws/HoReN-paper-reproduction` clone,去掉嵌套 `.git` 后 vendored)

## 关键接口(以 stub 落地,签名见各文件 docstring)
- `editing.edit(model, memory) -> edited_model | edit_module`(输出格式 full state_dict vs delta/side-module TBD,决定 hot-swap 路径)
- `generate.generate(query, *, model, buffer, rag_hits, with_rag=True) -> str`
- `memory.router.route(item) -> "edit"|"rag"`(atomic ∧ internalize ∧ stable → edit,否则 rag;轴是 SHAPE 不是 category)
- `memory.dedup.classify(candidate, consolidated) -> "duplicate"|"supersede"|"new"`(在 consolidation 时跑)
- `memory.consolidate.run_pass(trigger) -> n_written`(UI 计数器)
- `memory.prompt.build_prompt(query, buffer, rag_hits) -> messages`(SYSTEM + 固定 RAG 窗口[buffer 段 + docs 段] + history + query;窗口恒在、内容可空)
