# v1.4 — query-span pooling 可调参数 + 自动扫 60%/40%

## 计划（开发前）

### 背景
HoReN chat 路径(Plan B v0.3)的 retrieval key 对 **query-span**(用户问句 token 段,排除 scaffold)做**写死的 flat mean**(`editor._pool_span`)。英文真样本实验显示 JQ 各事实 key 挤在 ~0.88 cosine 窄锥(DIRECT 0.896 / PARA 0.879 / JQ-内 NEG 地板 0.880),三档几乎不可分、locality 破。怀疑「整段 flat mean 摊平后段语义 token」是元凶之一。本轮把 query-span 内部池化做成**可调旋钮**便于调试,默认 flat(不动线上),自动测 60%/40%。

> 不动 raw/eval 路径的 `query_selection_strategy`(`_select_query`,对整段 prompt)。本轮新增的是 chat 路径对 query-span 的独立旋钮。

### 修改（全部 additive，默认 flat = 行为不变）
- **`third_party/horen/src/models/horen/editor.py`**
  - 新增 module-level 纯函数 `pool_span_rows(layer_input, start, end, strategy="flat")`(query-span 池化单一来源):`flat`/`mean`/`avg` → 全段均值;`last`/`last_token`/`last_prompt_token` → 末行;`last_<p>_perc`(正则 `^last_(\d+(?:\.\d+)?)_perc(?:_span_avg)?$`)→ span 内最后 `ceil(n*p/100)` 行均值;未知值 `ValueError`。
  - `_pool_span` 改薄包装 → `pool_span_rows(..., self.query_span_pool_strategy)`。`forward`(read-key)与 `keying.compute_key`(write-key)都经 `_pool_span`,读/写自动一致。
  - `__init__` 读 `self.query_span_pool_strategy = _cfg(config, "query_span_pool_strategy", "flat")`。
- **`third_party/horen/src/models/horen/horen_hparams.py`**:dataclass 加 `query_span_pool_strategy: str = "flat"`。
- **`third_party/horen/hparams/HOREN/llama3.1-8b.yaml`**:加 `query_span_pool_strategy: flat`(显式默认;调试改这行)。
- **`keying.py`**:不改(`compute_key` 经 `adapter._pool_span` 自动继承旋钮)。
- **`spikes/spike_v14_query_span_pool_sweep.py`**(新,measurement-only):import `pool_span_rows`,英文真样本 A0001–A0005 × 三 key-format 臂(cloze/stmt/ques)× 探针(DIRECT/PARA/JQ-NEG/generic-NEG),sweep `strategy ∈ {flat, last_40_perc, last_60_perc, last_80_perc, last}`,出 summary(DIRECT/PARA 均值、PARA 触发、JQ-内 NEG 地板、generic-NEG 地板、PARA−NEG margin)。

### 测试
- CPU 小单测:`pool_span_rows` 在合成 `[1,n,D]` 上 `flat==mean`、`last==末行`、`last_60_perc`=⌈n·0.6⌉ 末行、`last_40_perc`=⌈n·0.4⌉ 末行;未知值抛错;`flat` 与旧 `_pool_span` 数值一致(线上零改动)。
- hparam:缺字段→默认 flat;yaml 写 `last_60_perc`→adapter 接住。
- GPU sweep(inline 串行,自载 model 副本,不碰 server):跑通 5 strategy,看 60%/40% 是否拉开 PARA−NEG margin。
- 全程 `torch 2.8.0+cu128 / True`,无装包。

## 结果（开发后）

### 落地 & 测试
- `editor.pool_span_rows` + `_pool_span` 薄包装 + `__init__` 读 `query_span_pool_strategy`(默认 `flat`);hparams dataclass + yaml 加同名字段(默认 `flat`)。
- CPU 单测全过:`flat`/`mean`/`avg` == 旧 `_pool_span` 均值(**线上字节不变**)、`last`==末行、`last_{40,60,80}_perc`=⌈n·p⌉ 末行、clamp、未知值/越界 `ValueError`;dataclass 默认 `flat`、yaml 载入 `flat`。
- `torch 2.8.0+cu128 / True` 全程;sweep VRAM allocated 16.2G / max 16.3G(自载副本,未碰 server)。

### Sweep（英文真样本 A0001–A0005,阈值 0.85;headline = cloze 生产臂）
| strategy | DIRECT | PARA | PARAfire | JQNEG 地板 | JQNEGxfire | genNEG 地板 | **PARA−NEG margin** |
|---|---|---|---|---|---|---|---|
| flat(现状) | 0.896 | 0.879 | 5/5 | 0.880 | 5/40 | 0.782 | **−0.000** |
| last_40_perc | 0.767 | 0.726 | 0/5 | 0.712 | 0/40 | 0.622 | +0.014 |
| last_60_perc | 0.835 | 0.803 | 0/5 | 0.803 | 0/40 | 0.709 | +0.000 |
| last_80_perc | 0.868 | 0.846 | 2/5 | 0.845 | 0/40 | 0.754 | +0.001 |
| last | 0.350 | 0.244 | 0/5 | 0.511 | 0/40 | 0.460 | −0.267 |

(stmt 臂 margin 略正:flat +0.018 → 60% **+0.044**(全 sweep 最佳)→ 80% +0.034;ques 臂 flat 17/40 误触发最差,DIRECT 恒 1.0 自匹配。)

### 结论
- **intra-span 池化 % 不是杠杆。** cloze 生产臂的 PARA−NEG margin 在 flat/40/60/80/last 全程 ≈ 0(最高 +0.014 = 噪声,且那点 PARA 0/5 根本不触发)。没有任何 % 能把 PARA 从 JQ-NEG 地板里拉开。
- **% 只是整体平移锥**:取越少 token,DIRECT 与 NEG **一起**等量下沉(DIRECT 0.896→0.835→0.767→0.350,NEG 地板 0.880→0.803→0.712→0.511),PARA 与 NEG 始终黏在一起 →**等价于动阈值**,不是分离信号。`last_80`/`60` 的「0 误触发」是假象:整锥沉到 0.85 下,PARA 也跟着不触发(2/5),margin 仍 ≈0。`last`-token 最差(全塌)。
- **坐实 [[horen-key-cone-collapse]] 诊断**:病在表示塌缩 + 单一固定阈值,**不在池化算子,也不在 key 文本**。真杠杆仍是 contrast-out 共享分量(codebook mean-center)/ 按分布标阈值。
- **线上默认保持 `flat` 不动**;旋钮留作调试入口。是否换默认/下一步动表示层 → 待用户定。
